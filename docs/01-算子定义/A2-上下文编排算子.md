# A2-上下文编排算子

## 1. 当前实现

| 项目 | 内容 |
|---|---|
| 注册名 | `video_context_orchestration` |
| 实现类 | `VideoContextOrchestrationOperation` |
| 源码 | `AscendDataForge/multimodal_data_processor/operations/filtering/video_context_orchestration_operation.py` |
| 基类 | `FilteringOperation` |
| operation_type | `OperationType.FILTERING` |

A2 在 `filter(dataset)` 中读取 `segments`，按模式生成编排后的 segments。默认 `output_key="segments"`，因此会覆盖 row 中原有 `segments` 字段，同时写入完整结果和指标。

## 2. 支持模式

| 模式 | 行为 | 输出变化 |
|---|---|---|
| `MERGE_STITCH` | 将时间间隔小、合并跨度不超过阈值、且相邻任一片段过短的 segments 放入同一 stitch group | 给每段写 `stitch_group_id`，清空 `context_window_refs` 与 `cleanup_flags` |
| `WINDOW_ATTACH` | 为每个 anchor segment 找到固定秒级窗口内的相邻 segment | 写 `context_window_refs`，`stitch_group_id=None` |
| `FILTER_HINT` | 按显式规则标记过短、空白文本、近重复文本 | 写 `cleanup_flags`，不删除 segment |

A2 不做语义事件识别、caption、summary、event tree、action graph 或 query scaffold。

## 3. 输入字段

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `segments_key` | `segments` | 输入分段字段 |
| `output_key` | `segments` | 输出分段字段 |
| `result_key` | `orchestration_result` | 完整结果字段 |

每个输入 segment 至少需要：

- `segment_id`
- `timestamp_range.start_sec`
- `timestamp_range.end_sec`
- `duration_sec`

`FILTER_HINT` 还会读取可选的 `text` 字段，用于判断 `blank_text` 和 `near_duplicate`。

## 4. 输出字段

A2 写回：

- `row[output_key]`：编排后的 segments，默认 `segments`
- `row[result_key]`：`orchestration_result`
- `row["orchestration_mode"]`
- `row["orchestration_metrics"]`

`orchestration_result` 包含：

- `orchestration_mode`
- `global_metrics.total_segments`
- `segments`

A2 追加到 segment 的字段：

- `stitch_group_id`
- `context_window_refs`
- `cleanup_flags`

当前实现不会写 `orchestration_reason`，也不会主动改写 `status`。

## 5. 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `orchestration_mode` | `MERGE_STITCH` | `MERGE_STITCH`、`WINDOW_ATTACH`、`FILTER_HINT` |
| `merge_gap_sec` | `0.5` | 可 stitch 的最大相邻间隔 |
| `max_merge_span_sec` | `8.0` | stitch group 的最大跨度 |
| `context_window_sec` | `2.0` | 上下文窗口秒数 |
| `max_context_segments` | `4` | 单个 anchor 最多保留的上下文引用数 |
| `min_valid_duration_sec` | `1.0` | 判定过短片段的阈值 |

## 6. 模式细节

### MERGE_STITCH

只在同时满足以下条件时把当前 segment 合入当前 group：

- 与上一段间隔 `gap_sec <= merge_gap_sec`
- 合并后跨度 `merged_span_sec <= max_merge_span_sec`
- 上一段或当前段的 `duration_sec < min_valid_duration_sec`

### WINDOW_ATTACH

对每个 segment，以 `[start_sec - context_window_sec, end_sec + context_window_sec]` 为窗口，收集时间重叠的其他 segment id，并截断到 `max_context_segments`。

### FILTER_HINT

当前规则：

- `too_short`：`duration_sec < min_valid_duration_sec`
- `blank_text`：`text` 为空或全空白
- `near_duplicate`：当前 `text` 与上一段 `text` 忽略大小写后相同，且间隔不超过 `merge_gap_sec`

## 7. 最小示例

```python
from multimodal_data_processor.core.operation_registry import OperationRegistry

op = OperationRegistry.create_operation(
    "video_context_orchestration",
    {"orchestration_mode": "FILTER_HINT", "min_valid_duration_sec": 1.0},
)
result_dataset = op.execute(dataset)
```
