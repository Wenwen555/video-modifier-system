# 一致性回修算子（A9）

## 1. 当前实现

| 项目 | 内容 |
|---|---|
| 注册名 | `video_consistency_revision` |
| 实现类 | `VideoConsistencyRevisionOperation` |
| 源码 | `AscendDataForge/multimodal_data_processor/operations/augmentation/video_consistency_revision_operation.py` |
| 基类 | `VideoAugmentationOperation` |
| operation_type | `OperationType.AUGMENTATION` |

A9 读取 `qae_triplets`、可选 `scaffold_units` 和 `qa_drafts`，生成 `revisions`。它不会直接改写 `qae_triplets`；A10 会在筛查时临时应用 `revisions[].after_fields`。

## 2. 支持模式

| 模式 | 实际行为 |
|---|---|
| `FIELD_ALIGN` | 如果 answer 与 question 相同，且 scaffold summary 可用，则把 answer 修为 scaffold summary |
| `EVIDENCE_PATCH` | 从 scaffold 重新构建 evidence refs，并在不同于 triplet 时输出替换建议 |
| `QUESTION_REWRITE` | 当 question 长度小于 3 且 scaffold 可用时，用规则问题替换 |
| `DRAFT_RECONCILE` | 如果 triplet 缺失 `source_qa_draft_id`，从 scaffold 的 `source_qa_draft_ids` 恢复 |

## 3. 输入字段

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `qae_triplets_key` | `qae_triplets` | 必需 |
| `scaffold_units_key` | `scaffold_units` | 可选，但部分模式需要 |
| `qa_drafts_key` | `qa_drafts` | `DRAFT_RECONCILE` 使用 |

`execute()` 会要求 `qae_triplets` 非空。

## 4. 输出字段

A9 写回：

- `revisions`
- `revision_result`
- `revision_mode`
- `revision_metrics`

每条 revision 包含：

- `id`
- `revision_id`
- `target_triplet_id`
- `revision_action`
- `before_fields`
- `after_fields`
- `reason`

`revision_result.global_metrics` 包含：

- `total_triplets`
- `total_revisions`

## 5. 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `output_key` | `revisions` | 输出字段 |
| `result_key` | `revision_result` | 完整结果字段 |
| `revision_mode` | `FIELD_ALIGN` | 修订模式 |
| `max_revision_rounds` | `1` | 参与扫描的 triplet 数量倍数；当前实现不会多轮重复改写同一 triplet |

## 6. 最小示例

```python
from multimodal_data_processor.core.operation_registry import OperationRegistry

op = OperationRegistry.create_operation(
    "video_consistency_revision",
    {"revision_mode": "EVIDENCE_PATCH"},
)
result_dataset = op.execute(dataset)
```
