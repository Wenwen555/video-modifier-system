# A3-采样算子

## 1. 当前实现

| 项目 | 内容 |
|---|---|
| 注册名 | `video_sampling` |
| 实现类 | `VideoSamplingOperation` |
| 源码 | `AscendDataForge/multimodal_data_processor/operations/augmentation/video_sampling_operation.py` |
| 基类 | `VideoAugmentationOperation` |
| operation_type | `OperationType.AUGMENTATION` |

A3 读取视频路径和 `segments`，输出 `samples`、`sampling_result` 和 `sampling_metrics`。如果 `save_sample_frames=True`，会用 OpenCV 从原视频抽帧落盘，并在 sample 中写入 `frame_path`。

## 2. 支持模式

| 模式 | 行为 | 依赖 |
|---|---|---|
| `CENTER` | 每段取中心帧 | `segments` + fps |
| `UNIFORM` | 每段均匀取 `num_frames` 个帧；若设置 `target_fps` 则按目标 fps 计算数量 | `segments` + fps |
| `HEAD_TAIL` | 每段取首帧和尾帧；单帧段只取首帧 | `segments` + fps |
| `STATE` | 在首尾各内缩 `padding_frames` 后取状态帧 | `segments` + fps |
| `SEMANTIC_AWARE` | 基于 `semantic_frame_features` 做最大语义距离多样性选择 | row 中必须有 `semantic_frame_features` |
| `MOTION_AWARE` | 使用同一组 `semantic_frame_features` 做按阈值间隔的多样性选择 | row 中必须有 `semantic_frame_features` |

`SEMANTIC_AWARE` 和 `MOTION_AWARE` 已实现，但不会自动提取 frame features；缺少 `semantic_frame_features` 会直接报错。

## 3. 输入字段

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `video_key` | `video_path` | 视频路径字段 |
| `segments_key` | `segments` | 输入分段字段 |

每个 segment 需要可解析出帧范围。优先使用：

- `timestamp_range.start_sec`
- `timestamp_range.end_sec`
- row 中的 `video_fps`、`fps` 或 `shot_fps`

若 fps 不显式提供，则尝试从 segment 的 `frame_range` 与 `timestamp_range` 反推。只有在 `save_sample_frames=True` 且仍不能解析 fps 时，才会尝试读取视频元数据。

## 4. 输出字段

A3 写回：

- `samples`
- `sampling_result`
- `sampling_metrics`

每个 sample 包含：

- `id`
- `sample_id`
- `segment_id`
- `timestamp_sec`
- `frame_index`
- `sample_type`
- `frame_path`：仅在 `save_sample_frames=True` 时写入

`sampling_result.global_metrics` 包含：

- `total_segments`
- `total_samples`
- `fps`
- `sample_frames_saved`

当前 sample 不写 `sampling_mode`、`relative_position`、`source_ids`、`confidence` 或 `status`。

## 5. 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `sampling_mode` | `CENTER` | 支持 6 种模式 |
| `num_frames` | `4` | `UNIFORM` 和自适应模式的目标数量 |
| `padding_frames` | `2` | `STATE` 模式首尾内缩帧数 |
| `target_fps` | `None` | `UNIFORM` 模式按目标 fps 计算采样数 |
| `max_frames_per_segment` | `None` | 自适应模式的单段上限 |
| `semantic_model_name` | `clip_vit_base_patch32` | 当前仅用于模型名校验和路径记录 |
| `semantic_min_distance_threshold` | `0.12` | 自适应模式的最小距离阈值 |
| `save_sample_frames` | `False` | 是否落盘采样帧 |
| `frame_output_dir` | `/data/wenjt/vid-ops/artifacts/sample_frames` | 采样帧输出目录 |
| `frame_path_key` | `frame_path` | sample 中保存帧路径的字段名 |
| `frame_image_format` | `jpg` | 输出图片格式 |
| `overwrite_sample_frames` | `False` | 已存在图片时是否覆盖 |

当前没有 `budget_frames`、`motion_threshold` 或 `strict` 参数。

## 6. 保存采样帧

当 `save_sample_frames=True`：

1. A3 按视频绝对路径计算 SHA1 前缀，创建子目录。
2. 文件名格式为 `{sample_id}_{frame_index}.{frame_image_format}`。
3. 使用 OpenCV 定位并写出图片。
4. 写回 sample 的 `frame_path_key` 字段。

如果视频打不开、帧越界、OpenCV 不可用或图片写入失败，会抛出异常。

## 7. 最小示例

```python
from multimodal_data_processor.core.operation_registry import OperationRegistry

op = OperationRegistry.create_operation(
    "video_sampling",
    {"sampling_mode": "UNIFORM", "num_frames": 4, "save_sample_frames": True},
)
result_dataset = op.execute(dataset)
```
