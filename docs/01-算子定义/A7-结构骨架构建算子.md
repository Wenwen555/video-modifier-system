# 结构骨架构建算子（A7）

## 1. 当前实现

| 项目 | 内容 |
|---|---|
| 注册名 | `video_scaffold_construction` |
| 实现类 | `VideoScaffoldConstructionOperation` |
| 源码 | `AscendDataForge/multimodal_data_processor/operations/augmentation/video_scaffold_construction_operation.py` |
| 基类 | `VideoAugmentationOperation` |
| operation_type | `OperationType.AUGMENTATION` |

A7 从 `description_units`、`temporal_structures`、`text_signals` 和可选 `samples` 构造 `scaffold_units`。当前不读取 `qa_drafts` 或 `evidence_units`。

## 2. 支持模式

| 模式 | 行为 | 必需输入 |
|---|---|---|
| `NARRATION_CHUNK` | 按 temporal structure 汇总描述和文本信号 | `temporal_structures` 可为空，为空则输出空列表 |
| `EVENT_TREE` | 为每个 temporal structure 构造 root/leaf tree | `temporal_structures` |
| `ACTION_GRAPH` | 对 temporal structure 关联采样帧做动作识别并构图 | `temporal_structures` + `samples[].frame_path` |

当前没有 `QUERY_SCAFFOLD` 模式，也不直接构建 question-conditioned scaffold。

## 3. 输入字段

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `description_units_key` | `description_units` | 输入描述单元 |
| `temporal_structures_key` | `temporal_structures` | 输入时序结构 |
| `text_signals_key` | `text_signals` | 输入文本信号 |
| `samples_key` | `samples` | `ACTION_GRAPH` 使用 |
| `frame_path_key` | `frame_path` | `ACTION_GRAPH` 读取采样帧路径 |

`NARRATION_CHUNK` 会用 `temporal_structures` 中的 `segment_ids` 找到对应 description/text。`EVENT_TREE` 和 `ACTION_GRAPH` 在缺少 `temporal_structures` 时会报错；`ACTION_GRAPH` 还要求有带 frame path 的 samples。

## 4. 输出字段

A7 写回：

- `scaffold_units`
- `scaffold_result`
- `scaffold_mode`
- `scaffold_metrics`

公共字段：

- `id`
- `scaffold_id`
- `unit_type`
- `temporal_unit_id`
- `scaffold_role`
- `segment_ids`
- `scaffold_summary`

模式字段：

| 模式 | 额外字段 |
|---|---|
| `NARRATION_CHUNK` | `source_description_ids`、`source_text_signal_ids` |
| `EVENT_TREE` | `root_ref`、`leaf_refs`、`tree_nodes`、`tree_edges`、`composition_rule` |
| `ACTION_GRAPH` | `action_nodes`、`action_edges`、`action_scene_model`、`text_signals` |

`scaffold_result.global_metrics` 包含输入和输出数量：`total_description_units`、`total_temporal_structures`、`total_text_signals`、`total_samples`、`total_scaffold_units`。

## 5. 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `output_key` | `scaffold_units` | 输出字段 |
| `result_key` | `scaffold_result` | 完整结果字段 |
| `scaffold_mode` | `NARRATION_CHUNK` | scaffold 模式 |
| `scaffold_role` | `generation` | 写入 scaffold 的角色字段 |
| `max_children_per_unit` | `4` | 每个单元保留的描述/文本/动作数量上限 |
| `leaf_window_sec` | `10.0` | `EVENT_TREE` 叶子窗口长度 |
| `action_scene_model_name` | `human_action_vit` | 动作识别模型别名 |
| `action_scene_model_path` | `None` | 自定义动作识别模型路径 |
| `action_scene_top_k` | `1` | 每帧动作分类候选数 |
| `action_scene_device` | `None` | 推理设备，例如 `cuda:0` 或 `cpu` |

动作模型别名：

- `human_action_vit` -> `rvv-karma/Human-Action-Recognition-VIT-Base-patch16-224`
- `human_action_siglip` -> `prithivMLmods/Human-Action-Recognition`
- `human_pose_vit` -> `Harsha901/vit-human-pose-classification-model`

## 6. 最小示例

```python
from multimodal_data_processor.core.operation_registry import OperationRegistry

op = OperationRegistry.create_operation(
    "video_scaffold_construction",
    {"scaffold_mode": "EVENT_TREE", "leaf_window_sec": 10.0},
)
result_dataset = op.execute(dataset)
```
