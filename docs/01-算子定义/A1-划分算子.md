# A1-划分算子

## 1. 当前实现

| 项目 | 内容 |
|---|---|
| 注册名 | `video_partition` |
| 实现类 | `VideoPartitionOperation` |
| 源码 | `AscendDataForge/multimodal_data_processor/operations/augmentation/video_partition_operation.py` |
| 基类 | `VideoAugmentationOperation` |
| operation_type | `OperationType.AUGMENTATION` |

A1 在 `execute(dataset)` 中逐行读取视频路径和分段依据，写回 `segments`、`partition_result`、`partition_metrics` 和 `shot_fps`。

## 2. 支持模式

| 模式 | 实际依据 | 必需输入 | 说明 |
|---|---|---|---|
| `FIXED` | 固定秒级窗口 | `video_duration_sec` 参数或 row 字段 | 从 0 秒开始按 `chunk_duration_sec` 生成边界 |
| `SEMANTIC` | 相邻语义 embedding 的 cosine distance | row 中 `semantic_embeddings`，或自动调用 `compute_video_semantic_embeddings` | 使用 `semantic_distance_threshold` 与 `min_segment_duration_sec` 判定边界 |
| `SHOT` | shot score 阈值 | row 中 `shot_scores`，或自动调用 `compute_video_shot_scores` | 使用 `shot_threshold` 与 `shot_fps` 计算边界时间 |

当前实现不使用 `transcript_entries`，不实现 Jaccard 文本切分、`HYBRID` 模式、多阶段 merge 或 ffmpeg 片段导出。

## 3. 输入字段

默认读取：

| 字段 | 默认名 | 说明 |
|---|---|---|
| 视频路径 | `video_path` | 由 `video_key` 参数控制，缺失时报错 |
| 视频时长 | `video_duration_sec` | `FIXED` 模式必需，可来自 row 或参数 |
| 语义 embedding | `semantic_embeddings` | `SEMANTIC` 模式可预先提供 |
| 镜头分数 | `shot_scores` | `SHOT` 模式可预先提供 |
| 镜头分数 fps | `shot_fps` | 默认取参数 `shot_fps=1.0` |

`semantic_embeddings` 中每个元素应包含 `start_sec`、`end_sec` 和 `embedding`。`shot_scores` 是按 `shot_fps` 对齐的一维分数序列。

## 4. 输出字段

A1 写回：

| 字段 | 说明 |
|---|---|
| `segments` | 分段列表 |
| `partition_result` | 完整分段结果，含 `video_path`、`partition_mode`、`global_metrics`、`partition_details`、`segments` |
| `partition_metrics` | 当前等于 `partition_result["global_metrics"]`，包含 `total_segments` |
| `shot_fps` | 实际使用的 shot fps |

每个 `segment` 包含：

- `id`
- `segment_id`
- `timestamp_range.start_sec`
- `timestamp_range.end_sec`
- `frame_range.start_frame`
- `frame_range.end_frame`
- `duration_sec`
- `partition_mode`
- `boundary_type`
- `status`

`boundary_type` 首段为 `START`；后续段按模式为 `FIXED_WINDOW`、`SEMANTIC_SHIFT` 或 `SHOT_CHANGE`。

当前实现不会为 segment 写入 `source_ids`、`confidence`、`operator_name`、`operator_version`、`resource_refs` 或 `processing_time_ms`。

## 5. 参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `video_key` | `video_path` | 输入视频路径字段名 |
| `partition_mode` | `FIXED` | `FIXED`、`SEMANTIC`、`SHOT` |
| `chunk_duration_sec` | `5.0` | 固定窗口长度 |
| `video_duration_sec` | `None` | 固定分段时的视频总时长 |
| `semantic_model_name` | `cosmos_embed1_224p` | 当前映射到 `nvidia/Cosmos-Embed1-224p` |
| `semantic_distance_threshold` | `0.35` | 语义边界阈值 |
| `semantic_window_sec` | `4.0` | 自动抽 embedding 的滑窗秒数 |
| `shot_model_name` | `transnetv2` | 当前映射到 `magnusdtd/TransNetV2` |
| `shot_threshold` | `0.55` | 镜头边界阈值 |
| `shot_fps` | `1.0` | shot score 对应帧率 |
| `min_segment_duration_sec` | `2.0` | 最小边界间隔 |

## 6. 常见错误

- `video_key` 指向的路径字段缺失：`Missing video path.`
- `FIXED` 模式缺少 `video_duration_sec`。
- `SEMANTIC` 模式无法取得 `semantic_embeddings`。
- `SHOT` 模式无法取得 `shot_scores`。
- 传入未支持的 mode 或模型名。

## 7. 最小示例

```python
from multimodal_data_processor.core.operation_registry import OperationRegistry

op = OperationRegistry.create_operation(
    "video_partition",
    {"partition_mode": "FIXED", "video_duration_sec": 12.0, "chunk_duration_sec": 5.0},
)
result_dataset = op.execute(dataset)
```
