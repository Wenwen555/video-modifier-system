# 高自由度视频算子系统总览

## 1. 文档口径

本页按 AscendDataForge 当前源码说明视频算子系统。实现中的算子不是独立的 `VideoPacket` 类，也不是 `process(instance)` 风格接口；它们继承 `VideoAugmentationOperation` 或 `FilteringOperation`，通过 `MultimodalDataset.map(...)` 对每条 row 字典追加或覆盖指定字段。

当前 A 编号主链覆盖 10 个已注册视频算子：

`A1 划分 -> A2 上下文编排 -> A3 采样 -> A4 多粒度描述 -> A5 时序结构 -> A6 文本辅证 -> A7 结构骨架 -> A8 QAE 物化 -> A9 一致性回修 -> A10 质量筛查`

另有已注册但不纳入 A 编号主链的通用视频算子：

- `video_loading`：加载本地视频文件为 `MultimodalDataset`
- `video_saving`：保存数据集
- `video_resize_aspect_ratio`：通过黑边调整视频宽高比

## 3. 当前数据流

当前实现的常用字段流如下：

```text
video_path / video_duration_sec / semantic_embeddings / shot_scores
  -> A1: segments, partition_result, partition_metrics
  -> A2: segments, orchestration_result, orchestration_metrics
  -> A3: samples, sampling_result, sampling_metrics
  -> A4: description_units, description_result, description_metrics
  -> A5: temporal_structures, temporal_result, temporal_metrics
  -> A6: text_signals, text_evidence_result, text_evidence_metrics
  -> A7: scaffold_units, scaffold_result, scaffold_metrics
  -> A8: qae_triplets, materialization_result, materialization_metrics
  -> A9: revisions, revision_result, revision_metrics
  -> A10: quality_records, quality_result, quality_metrics
```

A4、A5、A6 在工程上可以按数据可用性和预算并行或重排，但 A4 的 `EVENT_DESCRIPTION` 需要 `temporal_structures`，A5 需要 `segments` 和 `samples`，A6 需要 `segments` 和 `samples`。

## 4. 实现边界

以下能力在旧设计文档中出现过，但当前 A1-A10 源码没有作为已实现模式落地：

- A1 不实现 `HYBRID`、跨模态联合边界判定、ffmpeg 片段物化导出，也不使用 `transcript_entries` 做 Jaccard 语义切分。
- A3 的自适应模式已实现为 `SEMANTIC_AWARE` 和 `MOTION_AWARE`，但二者都依赖 row 中已有的 `semantic_frame_features`，不会自动抽取运动特征。
- A4 不实现 `MULTI_SENTENCE_SUMMARY` 模式。
- A5 不实现 `MULTIHOP_BRIDGE` 模式。
- A7 不实现 `QUERY_SCAFFOLD` 模式，也不直接读取 `qa_drafts` 或 `evidence_units`。
- A8 支持 `QA_FIRST_GROUNDING`，但默认 `use_llm_client=True` 且 `require_llm_client=True`；没有 LLM 客户端时需要显式关闭或放宽该要求。
- A10 不实现 `SCORE_GATE` 或 `VALUE_RANK` 模式；当前只有规则过滤和层级分配。

## 5. 设计目标与实现关系

系统目标仍是为视频数据构造提供可追溯的中间结果，服务 grounded QA 或 post-training 数据准备。但当前代码层的可靠口径是“row 字段协议 + 注册算子 + 结果字段”，不是一个强制类型化的统一 `VideoPacket` 对象。各单算子页面给出实际输入输出与参数，出现冲突时以源码为准。
