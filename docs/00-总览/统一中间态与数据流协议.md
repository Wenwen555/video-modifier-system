# 统一中间态与数据流协议

## 1. 当前实现口径

当前源码没有实现独立的 `VideoPacket`、`OperatorConfig` 或 `Selector` 类型。视频算子实际消费的是 `MultimodalDataset` 中的 row 字典，并通过参数中的 `*_key` 指定读取和写回字段。

因此，本文档把统一中间态定义为“row 字段协议”：只要一条 row 包含上游算子产出的字段，下游就可以继续消费。

## 2. 顶层字段

当前 A1-A10 主链常用字段如下：

| 字段 | 生产算子 | 说明 |
|---|---|---|
| `video_path` | 上游数据或 `video_loading` | 默认视频路径字段 |
| `video` | 上游数据 | A6 读取 `subtitle_uri`、`asr_uri` 的默认容器 |
| `segments` | A1、A2 | 视频片段列表；A2 默认覆盖该字段 |
| `partition_result` | A1 | 分段完整结果 |
| `partition_metrics` | A1 | 分段指标，当前含 `total_segments` |
| `samples` | A3 | 采样帧元数据列表 |
| `sampling_result` | A3 | 采样完整结果 |
| `sampling_metrics` | A3 | 采样指标 |
| `description_units` | A4 | 多粒度描述单元 |
| `description_result` | A4 | 描述完整结果 |
| `temporal_structures` | A5 | 时间结构单元 |
| `temporal_result` | A5 | 时间结构完整结果 |
| `text_signals` | A6 | OCR、字幕或 ASR 文本信号 |
| `text_evidence_result` | A6 | 文本证据完整结果 |
| `scaffold_units` | A7 | scaffold 单元 |
| `scaffold_result` | A7 | scaffold 完整结果 |
| `qae_triplets` | A8 | Q/A/E 三元组 |
| `materialization_result` | A8 | QAE 物化完整结果 |
| `revisions` | A9 | 修订记录 |
| `revision_result` | A9 | 修订完整结果 |
| `quality_records` | A10 | 质量筛查记录 |
| `quality_result` | A10 | 筛查完整结果 |

## 3. 最小对象结构

### 3.1 Segment

由 A1 生成，A2 会追加编排字段。

最小字段：

- `id`
- `segment_id`
- `timestamp_range.start_sec`
- `timestamp_range.end_sec`
- `frame_range.start_frame`
- `frame_range.end_frame`
- `duration_sec`
- `partition_mode`
- `boundary_type`
- `status`

A2 追加字段：

- `stitch_group_id`
- `context_window_refs`
- `cleanup_flags`

### 3.2 Sample

由 A3 生成。

- `id`
- `sample_id`
- `segment_id`
- `timestamp_sec`
- `frame_index`
- `sample_type`
- `frame_path`：仅在 `save_sample_frames=True` 时写入，字段名由 `frame_path_key` 控制

### 3.3 Description Unit

由 A4 生成。

- `id`
- `description_id`
- `description_unit_id`
- `segment_id`
- `description_text`
- `description_scope`
- `granularity`
- `anchor_refs`

### 3.4 Temporal Structure

由 A5 生成。

- `id`
- `temporal_unit_id`
- `segment_id`
- `segment_ids`
- `timestamp_range`
- `structure_type`
- `summary`

### 3.5 Text Signal

由 A6 生成。

- `id`
- `text_signal_id`
- `signal_id`
- `segment_id`
- `timestamp_range`
- `text_type`
- `text`

### 3.6 Scaffold Unit

由 A7 生成，字段随模式变化。

公共字段：

- `id`
- `scaffold_id`
- `unit_type`
- `temporal_unit_id`
- `scaffold_role`
- `segment_ids`
- `scaffold_summary`

模式字段：

- `NARRATION_CHUNK`：`source_description_ids`、`source_text_signal_ids`

### 3.7 QAE Triplet

由 A8 生成。

- `id`
- `triplet_id`

- `triplet_type`

- `llm_materialization_analysis`：可选

由 A9 生成。

- `id`
- `revision_id`
- `target_triplet_id`
- `revision_action`

- `after_fields`

### 3.9 Quality Record

由 A10 生成。

- `id`
- `quality_id`

- `rank_in_video`

## 4. Evidence References

A8 当前输出的 `evidence_refs` 是分 modality 的字典：

```json
{
  "text": {
    "description_ids": [],
    "text_signal_ids": [],
    "evidence_unit_ids": []
  },
  "image": {
    "frame_refs": [],
    "sample_ids": [],
    "frame_paths": []
  },
  "temporal": {
    "temporal_unit_ids": []
  }
}
```

A10 使用 `evidence_ref_count(...)` 统计上述引用数量，并据此分配：

- `qa`：无有效证据或证据数量不足
- `grounded_qa`：恰有 1 个有效证据引用
- `strong_grounded_qa`：超过 1 个证据引用，或 `grounding_level == "strong_grounded"`

## 5. 写入规则

当前实现以“写回 row 字段”为准，不是严格追加式对象存储：

- A1 写入 `segments`。
- A2 默认以 `output_key="segments"` 覆盖上游 `segments`，同时保存 `orchestration_result`。
- A3-A8 写入各自的输出列表和 result/metrics 字段。
- A9 写入 `revisions`，不直接改写 `qae_triplets`。
- A10 写入 `quality_records`，筛查时会在内存中应用 revision 后的字段。

## 6. 错误处理

当前 A1-A10 没有统一的 `strict` 参数。缺少必需字段、模式不支持、模型依赖不可用或文件不存在时，算子通常直接抛出 `ValueError` 或 `ImportError`。
