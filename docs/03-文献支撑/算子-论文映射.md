# 算子-论文映射

## 1. 使用说明

本文件说明当前 A1-A10 实现与代表性文献流程之间的关系。映射等级表达“设计来源强度”，不表示源码已经完整复现论文系统。

- `P2`：论文中存在较明确的核心环节，当前算子实现了可复用工程子集。
- `P1`：由多篇论文中的共同需求抽象而来，当前实现提供轻量或规则化版本。
- `P0`：主要服务工程闭环。

## 2. 总表

| 算子 | 等级 | 当前实现模式 | 主要文献 | 保留的核心能力 |
|---|---|---|---|---|
| A1 划分算子 | P1 | `FIXED`、`SEMANTIC`、`SHOT` | Panda-70M, TimeChat-Captioner | 初始时间片段划定 |
| A2 上下文编排算子 | P1 | `MERGE_STITCH`、`WINDOW_ATTACH`、`FILTER_HINT` | Panda-70M, Shot2Story | 低成本 stitch、上下文引用、显式清洗标记 |
| A3 采样算子 | P2 | `CENTER`、`UNIFORM`、`HEAD_TAIL`、`STATE`、`SEMANTIC_AWARE`、`MOTION_AWARE` | Video-ChatGPT, Adaptive Keyframe Sampling | 基础采样与 feature-aware 多样性选择 |
| A4 多粒度描述算子 | P1 | `FRAME_DESCRIPTION`、`EVENT_DESCRIPTION`、`SEGMENT_DESCRIPTION`、`CONTEXT_DESCRIPTION` | TimeChat-Captioner, Shot2Story, LongViTU | 多粒度 caption 描述单元 |
| A5 时序结构抽取算子 | P2 | `EVENT_WINDOW`、`EVENT_CHAIN` | Grounded QA, Grounded Multi-Hop VideoQA, LongViTU | 时间窗和事件链结构 |
| A6 文本辅证抽取算子 | P1 | `OCR`、`SUBTITLE_PARSE`、`ASR_ALIGN` | MME-VideoOCR, LongVideoBench | 文本证据信号抽取与按 segment 对齐 |
| A7 结构骨架构建算子 | P1 | `NARRATION_CHUNK`、`EVENT_TREE`、`ACTION_GRAPH` | Grounded QA, Grounded Multi-Hop VideoQA, LongViTU | narration、tree、graph 三类 scaffold |
| A8 QAE 物化算子 | P2 | `JOINT_SYNTHESIS`、`DRAFT_BINDING`、`ANSWER_VERIFY_FILL`、`QA_FIRST_GROUNDING` | Grounded QA, Grounded Multi-Hop VideoQA, LongViTU | 从 scaffold/草稿/证据生成 triplet |
| A9 一致性回修算子 | P1 | `FIELD_ALIGN`、`EVIDENCE_PATCH`、`QUESTION_REWRITE`、`DRAFT_RECONCILE` | LongViTU, VDC-Agent | 生成可审计 revision record |
| A10 质量筛查算子 | P2 | `RULE_FILTER`、`TIER_ASSIGN` | LongViTU, Omni-Judge, MVQA-68K | 证据完整性筛查与层级分配 |

## 3. 当前强支撑算子

- A3：采样是视频理解数据构造的稳定基础环节。
- A5：时间窗和事件链直接服务 temporal grounding。
- A8：Q/A/E triplet 物化是 grounded QA 数据构造核心。
- A10：最终过滤和层级分配是训练数据出站前必要步骤。

## 4. 实现边界提醒

- A5 当前没有 `MULTIHOP_BRIDGE`。
- A7 当前没有 `QUERY_SCAFFOLD`。
- A10 当前没有 `SCORE_GATE` 和 `VALUE_RANK`。
- 这些能力可以作为后续扩展方向，但不应在当前文档中描述为已实现。
