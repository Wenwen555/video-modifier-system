# 文献支撑与最新趋势总结

## 1. 范围

本文件总结当前 10 个视频算子设计与近期视频数据构造/grounded QA 文献的对应关系。实现状态以 AscendDataForge 当前源码为准；文献趋势用于解释为什么这些算子合理，不等同于声明源码完整复现论文系统。

## 2. 总体结论

### 2.1 已有较强近期支撑的算子

- A3 采样算子
- A5 时序结构抽取算子
- A8 QAE 物化算子
- A10 质量筛查算子

### 2.2 以组合形式出现、但工程上必要的算子

- A1 划分算子
- A2 上下文编排算子
- A4 多粒度描述算子
- A6 文本辅证抽取算子
- A7 结构骨架构建算子
- A9 一致性回修算子

这些算子在论文中常被折叠进数据处理 pipeline。当前实现把它们拆成可注册、可测试的操作单元。

## 3. 当前研究趋势与实现对应

### 3.1 数据构造更偏向结构或证据先行

近期 grounded QA 数据构造常先组织 narration chunks、event structures、action graphs 或 timestamped evidence，再生成 QA。当前对应实现为：

- A5 `EVENT_WINDOW` / `EVENT_CHAIN`
- A7 `NARRATION_CHUNK` / `EVENT_TREE` / `ACTION_GRAPH`
- A8 `JOINT_SYNTHESIS`

### 3.2 多粒度描述是可复用中间层

当前 A4 不是单一 caption 出口，而是支持：

- `FRAME_DESCRIPTION`
- `EVENT_DESCRIPTION`
- `SEGMENT_DESCRIPTION`
- `CONTEXT_DESCRIPTION`

这些描述会被 A5、A7、A8 复用。

### 3.3 文本信号仍是高价值辅证

字幕、ASR、OCR 常常决定答案是否精确。当前 A6 已实现：

- OCR：基于采样帧或预提取文本。
- Subtitle：读取 `subtitle_uri`。
- ASR：读取 `asr_uri`。

### 3.4 QA-first 是兼容路径，不是当前 A7 主模式

A8 已实现 `QA_FIRST_GROUNDING`，可以根据 QA 草稿中的 clip/segment/time 信息回挂证据。但 A7 当前没有 `QUERY_SCAFFOLD`，因此文档中不应把 query scaffold 写成 A7 已实现模式。

### 3.5 回修与筛查是必要出口

当前 A9 输出 revision records，A10 输出 quality records。二者提供可审计的出站链路：

- A9 记录 `before_fields` / `after_fields`。
- A10 输出 `decision`、`reason`、`scores` 和 `assigned_dataset_tier`。

## 4. 当前系统最对应的文献模式

| 文献模式 | 当前实现对应 |
|---|---|
| Narration Chunk to `(Q, A, T)` | A6 + A5 + A7 `NARRATION_CHUNK` + A8 |
| Action Scene Graph to Triplets | A3 保存帧 + A5 + A7 `ACTION_GRAPH` + A8 |
| Hierarchical Video Tree to Timestamped QA | A5 + A7 `EVENT_TREE` + A8 + A9 |
| QA-first / post-hoc grounding | 外部 `qa_drafts` + A8 `QA_FIRST_GROUNDING` |

## 5. 当前不应过度声明的能力

- 不声明 A1 已实现文本 transcript Jaccard 语义切分。
- 不声明 A5 已实现多跳桥接模式。
- 不声明 A7 已实现 query scaffold。
- 不声明 A10 已实现 value ranking 或通用 score gate。
- 不声明所有算子共享强类型 `VideoPacket`；当前是 row 字段协议。
