# 文献支撑与最新趋势总结

## 1. 范围
本文件总结截至 `2026-04-08` 可用于支撑当前 10 算子设计的近期文献脉络，并额外列出若干值得在汇报中单独展示的新趋势。

## 2. 总体结论

### 2.1 已有较强近期支撑的算子
- A3 采样算子
- A5 时序结构抽取算子
- A8 QAE共同合成算子
- A10 质量筛查算子

### 2.2 以组合形式出现、但已足够合理的算子
- A1 划分算子
- A2 上下文编排算子
- A4 多粒度描述算子
- A6 文本辅证抽取算子
- A7 结构骨架构建算子
- A9 一致性回修算子

这些算子并不是设计有问题，而是当前文献更倾向于把相关能力折叠进更大的数据构造 pipeline，而不是把它们单独命名为算子。

其中需要特别收紧理解的是 A2：它更适合作为“低成本清洗与简单编排”来解释，而不宜继续扩张成语义 pack 层。

## 3. 当前研究专注的五个方向

### 3.1 数据构造论文更偏向“结构骨架优先”
越来越多工作不是直接从原视频端到端生成问答，而是先构造：

- narration chunks
- hierarchical trees
- action scene graphs
- timestamped event structures

再从这些结构中派生数据。

### 3.2 A4 这类描述层正在向“多粒度复用”演化
在当前更成熟的链路里，描述并不是只出现一次，而是会在多个粒度反复使用：

- frame-level description
- event-level description
- segment-level description
- context-level summary

这也是为什么当前系统把 A4 定义为“多粒度描述算子”，而不是“粗 caption 算子”。

### 3.3 时间结构比空间框更早进入数据构造主链
对于数据构造任务，当前更成熟的是：

- `Q + A + temporal evidence`

而不是：

- `Q + A + spatial boxes`

因此，时间窗、事件链和多跳桥接往往先成为主链，而空间框更多仍在推理或增强链里。

### 3.4 共同合成比 post-hoc grounding 更接近主流数据构造
当前很多数据构造工作更接近：

- `S -> (Q, A, E)`

其中 `S` 是某种结构骨架。

这意味着 Question、Answer 和 Evidence 共享上游结构，而不是“先问、再回视频查证”。

### 3.5 回修与 refinement 仍然是必要环节
即使采用共同合成，当前文献也很少直接接受首次生成结果。

常见做法包括：

- self-revision
- filtering
- refinement
- 人工抽检

## 4. 当前系统最对应的文献模式

### 4.1 Narration Chunk to `(Q, A, T)`
代表方向：

- Grounded Question-Answering in Long Egocentric Videos

核心思路：

- 先组织 narration chunks
- 再共同生成问题、答案和时间证据

### 4.2 Action Scene Graph to Triplets
代表方向：

- Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos

核心思路：

- 先构建图结构
- 再在图上生成 question-answer-evidence triplets

### 4.3 Hierarchical Video Tree to Timestamped QA
代表方向：

- LongViTU

核心思路：

- 先构建层级视频树
- 再在 subtree 上生成带证据的 QA
- 最后做 self-revision

### 4.4 Why Not Default to Post-hoc Grounding
代表方向：

- Pinpointing Trigger Moment for Grounded Video QA
- Video-in-the-Loop

核心思路：

- 这些方法很重要，但更偏推理和 challenge 场景
- 它们说明了 grounded QA 的另一大分支
- 但并不是当前数据构造主链的最常见模式
