# 文献支撑与最新趋势总结

## 1. 范围
本文件总结截至 `2026-03-30` 可用于支撑当前 10 算子设计的近期文献脉络，并额外列出若干值得在汇报中单独展示的 2025-2026 新趋势。

## 2. 总体结论

### 2.1 已有较强近期支撑的算子
- A3 采样算子
- A4 自适应采样算子
- A5 去重算子
- A7 空间聚焦算子
- A8 时序定位与变化摘要算子
- A9 标注生成算子
- A10 候选选择与质量裁决算子

### 2.2 支撑相对较弱但仍必要的算子
- A1 划分算子
- A2 片段清洗算子
- A6 文本抽取算子

这些算子并不是设计有问题，而是最新文献更倾向于把相关能力折叠进更大的 grounding、captioning 或 retrieval 框架中，而不是把它们单独命名为算子。

## 3. 当前研究专注的五个方向

### 3.1 自适应采样已经成为长视频与复杂视频理解的核心入口
从 Adaptive Keyframe Sampling 到 AdaRD-Key，再到 Video-in-the-Loop 和 Think with Grounding，近期工作都在强调：

- 均匀采样会稀释关键时刻
- 固定预算必须重新分配
- relevance 和 diversity 需要共同考虑

### 3.2 去重和压缩已经从工程优化升级为建模能力的一部分
STORM、V-CORE、SlowFast-LLaVA-1.5 和 Token Reduction via AOT 说明：

- 长视频 token 压缩不只是节省算力
- 压缩策略会直接影响时序信息是否保留下来

### 3.3 空间聚焦和时序定位正在合流
VideoRefer Suite、SpaceVLLM、VideoLoom、SlotVTG 强调对象级和区域级 grounding；LeAdQA、ED-VTG、TimeChat-Captioner、Think with Grounding 强调时间边界和关键时刻定位。

这两条线合起来，正好支撑 A7 和 A8 的并行设计。

### 3.4 标注生成正从 caption 走向 grounded caption、script 和结构化 QA
LongViTU、GROVE、VideoRefer Suite、TimeChat-Captioner、VDC-Agent 表明自动化数据构建已经不再局限于“看视频写一句话”。

### 3.5 质量裁决开始强调解释性和反思式评分
LongViTU、MVQA-68K、VDC-Agent、Omni-Judge 都显示，后验质量控制的趋势不是输出一个分数，而是：

- 细分维度
- 给出保留或拒绝原因
- 支持重试或修正

## 4. 最新的视频算子设计思路展示

### 4.1 Query-Conditioned Keyframe Sampler
代表方向：

- Adaptive Keyframe Sampling
- AdaRD-Key

核心思路：

- 采样不再只依赖视频本身
- 采样应该对问题或任务目标有感知

### 4.2 Keyframe + Narrative Interleaver
代表方向：

- Threading Keyframe with Narratives

核心思路：

- 关键帧之间需要文本桥接
- 否则视觉锚点之间的时间连续性会断裂

### 4.3 Span-Aware Token Reallocator
代表方向：

- Video-in-the-Loop

核心思路：

- 先低成本浏览全视频
- 再把更多预算集中到关键时间段

### 4.4 On-Demand Grounding
代表方向：

- Think with Grounding

核心思路：

- 推理过程中可以二次回看视频
- grounding 是动态触发的，而不是一次性完成

### 4.5 Structured Time-Aware Dense Captioner
代表方向：

- TimeChat-Captioner

核心思路：

- caption 不再只是单句摘要
- 而是带时间边界、多场景和结构化 schema 的描述

### 4.6 Joint Spatial-Temporal Grounder
代表方向：

- VideoLoom
- SpaceVLLM

核心思路：

- 空间定位和时间定位应该联合建模
- 对象、区域和时间段需要共同约束

### 4.7 Object-Centric Temporal Adapter
代表方向：

- SlotVTG

核心思路：

- 用更对象中心的表征替代纯帧级 token
- 提升细粒度时间定位和泛化能力

### 4.8 Self-Reflective Caption Judge
代表方向：

- VDC-Agent

核心思路：

- caption 生成、评分和修正形成闭环
- 质量控制不再是纯后处理
