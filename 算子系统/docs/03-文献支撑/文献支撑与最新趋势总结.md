# 文献支撑与最新趋势总结

## 1. 范围
本文件总结截至 `2026-04-07` 可用于支撑当前 9 算子设计的近期文献脉络，并额外列出若干值得在汇报中单独展示的 2025-2026 新趋势。

## 2. 总体结论

### 2.1 已有较强近期支撑的算子
- A3 采样算子
- A4 时序证据定位算子
- A5 空间证据聚焦算子
- A9 Grounded回修与质量筛查算子

### 2.2 以组合形式出现、但已足够合理的算子
- A1 划分算子
- A2 上下文编排算子
- A6 文本辅证抽取算子
- A7 草案生成算子
- A8 Query规范化算子

这些算子并不是设计有问题，而是最新文献更倾向于把相关能力折叠进更大的 grounding、agentic generation 或 retrieval 框架中，而不是把它们单独命名为算子。

## 3. 当前研究专注的五个方向

### 3.1 自适应采样已经成为长视频与复杂视频理解的核心入口
从 Adaptive Keyframe Sampling 到 AdaRD-Key，再到 Video-in-the-Loop 和 Think with Grounding，近期工作都在强调：

- 均匀采样会稀释关键时刻
- 固定预算必须重新分配
- relevance 和 diversity 需要共同考虑

这也是为什么当前系统把基础采样和自适应采样统一收敛进 `A3`，把它视为一个算子内部的模式谱系，而不是两个串行阶段。

### 3.2 时序证据定位正在从“通用前置”转向“query / draft 条件化回看”
Video-in-the-Loop、Can I Trust Your Answer?、LeAdQA、ED-VTG、Think with Grounding 等工作共同说明：

- 不是所有帧都值得进入后续答案生成
- 关键 span、变化边界和事件链必须先被显式找出
- 更有效的做法往往是先有 question 或草案，再让 temporal grounding 回到视频里缩小范围

这正好支撑了当前系统中 A4 的新默认语义：它是 `query-conditioned temporal revisit`，而不再默认被写成“无条件先行的时序池”。

### 3.3 空间聚焦和文本辅证正在成为被路由触发的并行证据分支
VideoRefer Suite、SpaceVLLM、VideoLoom、SlotVTG 强调对象级和区域级 grounding；MME-VideoOCR、LongVideoBench、TimeChat-Captioner 强调 OCR、字幕和 ASR 对问答可靠性的提升。

这两条线与时序证据一起，正好支撑 A4、A5、A6 的并行证据设计。当前系统进一步把它们写成：

- 先用 A4 缩小时序范围
- 再由 A5 和 A6 按 query bundle 预算回看取证

### 3.4 自动化数据构建正在从“直接生成”转向“草案 -> query -> grounding -> 回修”
Pinpointing Trigger Moment for Grounded Video QA、LongViTU、VDC-Agent、Think with Grounding 表明高质量数据构建越来越强调：

- 先形成低成本草案
- 再把草案拆成可执行 query
- 回到视频中找显式证据
- 最后根据证据修订答案，而不是保留最初草案

这正好对应当前系统中：

- A7 草案生成
- A8 Query规范化
- A4/A5/A6 回看取证
- A9 Grounded回修与质量筛查

### 3.5 Grounded 质量控制已经从“打分”转向“审查 + 回修 + 筛查”
LongViTU、MVQA-68K、VDC-Agent、Omni-Judge 以及近期 grounded QA 研究都显示，后验质量控制的趋势不是输出一个分数，而是：

- 细分维度
- 给出保留或拒绝原因
- 支持 evidence replay、答案回修或明确拒答

这也是为什么当前系统把“严格审查 Agent”收进 A9，而不是再单独开一个缺少稳定协议的新算子。

## 4. 最新的视频算子设计思路展示

### 4.1 Query-Conditioned Keyframe Sampler
代表方向：

- Adaptive Keyframe Sampling
- AdaRD-Key

核心思路：

- 采样不再只依赖视频本身
- 采样应该对问题或任务目标有感知

### 4.2 Structured Draft Generator
代表方向：

- Pinpointing Trigger Moment for Grounded Video QA
- LongViTU
- VDC-Agent

核心思路：

- 先得到粗 caption 或 QA 草案
- 再把整句草案拆成 claim-level 中间态

### 4.3 Query Normalizer and Budget Router
代表方向：

- Video-in-the-Loop
- Think with Grounding

核心思路：

- 自然语言问题不能原样直接送入 grounding
- query rewrite 和预算路由应被显式建模

### 4.4 Span-First Temporal Grounder
代表方向：

- Video-in-the-Loop
- Can I Trust Your Answer?

核心思路：

- 先缩小时序范围
- 再决定是否触发更贵的局部证据分支

### 4.5 Joint Spatial-Text Evidence Backfill
代表方向：

- SpaceVLLM
- VideoLoom
- TimeChat-Captioner

核心思路：

- 空间区域和文本辅证应作为并行补强分支
- 它们的触发应受同一个 query bundle 约束

### 4.6 Strict Audit and Repair Gate
代表方向：

- VDC-Agent
- Omni-Judge

核心思路：

- QA 生成、审查和修正形成闭环
- 质量控制不再是纯后处理
