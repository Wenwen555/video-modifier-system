# 文献支撑与最新趋势总结

## 1. 范围
本文件总结截至 `2026-03-30` 可用于支撑当前 9 算子设计的近期文献脉络，并额外列出若干值得在汇报中单独展示的 2025-2026 新趋势。

## 2. 总体结论

### 2.1 已有较强近期支撑的算子
- A3 采样算子
- A4 时序证据定位算子
- A5 空间证据聚焦算子
- A8 QA生成算子
- A9 Grounded核验与回修算子

### 2.2 以组合形式出现、但已足够合理的算子
- A1 划分算子
- A2 上下文编排算子
- A6 文本辅证抽取算子
- A7 问题规划算子

这些算子并不是设计有问题，而是最新文献更倾向于把相关能力折叠进更大的 grounding、agentic generation 或 retrieval 框架中，而不是把它们单独命名为算子。

## 3. 当前研究专注的五个方向

### 3.1 自适应采样已经成为长视频与复杂视频理解的核心入口
从 Adaptive Keyframe Sampling 到 AdaRD-Key，再到 Video-in-the-Loop 和 Think with Grounding，近期工作都在强调：

- 均匀采样会稀释关键时刻
- 固定预算必须重新分配
- relevance 和 diversity 需要共同考虑

这也是为什么当前系统把基础采样和自适应采样统一收敛进 `A3`，把它视为一个算子内部的模式谱系，而不是两个串行阶段。

### 3.2 时序证据定位已经成为生成高质量 QA 的关键前置
LeAdQA、ED-VTG、Video-in-the-Loop、Think with Grounding 和 TimeChat-Captioner 说明：

- 不是所有帧都值得进入问答生成
- 关键 span、变化边界和事件链必须先被显式找出
- 高质量 QA 越来越依赖“先定位，再提问”

### 3.3 空间聚焦和文本辅证正在成为并行证据分支
VideoRefer Suite、SpaceVLLM、VideoLoom、SlotVTG 强调对象级和区域级 grounding；MME-VideoOCR、LongVideoBench、TimeChat-Captioner 强调 OCR、字幕和 ASR 对问答可靠性的提升。

这两条线与时序证据一起，正好支撑 A4、A5、A6 的并行证据设计。

### 3.4 自动化数据构建正从“直接生成”转向“先规划，再生成”
LongViTU、VDC-Agent、VideoRefer Suite、Think with Grounding 表明高质量数据构建越来越强调：

- 先决定问什么
- 再决定用哪些证据回答
- 最后才输出自然语言 QA

这正好对应 A7 问题规划和 A8 QA生成的分层。

### 3.5 Grounded 核验开始强调解释性、回放和回修
LongViTU、MVQA-68K、VDC-Agent、Omni-Judge 都显示，后验质量控制的趋势不是输出一个分数，而是：

- 细分维度
- 给出保留或拒绝原因
- 支持 evidence replay、重试或修正

## 4. 最新的视频算子设计思路展示

### 4.1 Query-Conditioned Keyframe Sampler
代表方向：

- Adaptive Keyframe Sampling
- AdaRD-Key

核心思路：

- 采样不再只依赖视频本身
- 采样应该对问题或任务目标有感知

### 4.2 Keyframe + Narrative Interleaver
代表方向：

- Threading Keyframe with Narratives

核心思路：

- 关键帧之间需要文本桥接
- 否则视觉锚点之间的时间连续性会断裂

### 4.3 Span-Aware Token Reallocator
代表方向：

- Video-in-the-Loop

核心思路：

- 先低成本浏览全视频
- 再把更多预算集中到关键时间段

### 4.4 On-Demand Grounding
代表方向：

- Think with Grounding

核心思路：

- 推理过程中可以二次回看视频
- grounding 是动态触发的，而不是一次性完成

### 4.5 Structured Question Planner
代表方向：

- VDC-Agent
- LongViTU

核心思路：

- 先把证据结构转成问题蓝图
- 再控制题型、难度和答案形式

### 4.6 Structured Time-Aware Dense Captioner
代表方向：

- TimeChat-Captioner

核心思路：

- caption 不再只是单句摘要
- 而是带时间边界、多场景和结构化 schema 的描述

### 4.7 Joint Spatial-Temporal Grounder
代表方向：

- VideoLoom
- SpaceVLLM

核心思路：

- 空间定位和时间定位应该联合建模
- 对象、区域和时间段需要共同约束

### 4.8 Object-Centric Temporal Adapter
代表方向：

- SlotVTG

核心思路：

- 用更对象中心的表征替代纯帧级 token
- 提升细粒度时间定位和泛化能力

### 4.9 Self-Reflective QA Judge
代表方向：

- VDC-Agent

核心思路：

- QA 生成、评分和修正形成闭环
- 质量控制不再是纯后处理
