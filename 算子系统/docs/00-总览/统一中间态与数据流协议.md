# 统一中间态与数据流协议

## 1. 设计目标
统一中间态的目的是让所有算子都遵循同一套输入输出语言，避免出现以下问题：

- 每个算子定义自己的 JSON 结构，导致下游难以复用
- 同一视频在不同链路中反复解码和重复计算
- 最终数据无法回溯到具体片段、采样点、证据单元和骨架节点
- Q/A/E 三者脱钩，无法解释它们为何属于同一条样本

系统统一使用 `VideoPacket` 作为唯一中间态容器。

当前 `VideoPacket` 的默认服务目标是：

- 构造适合 `post-training / instruction tuning` 的高质量 LVLM QA 数据
- 兼容 `narration-first`、`structure-first`、`QA-first / post-hoc grounding` 三条主要数据主线
- 保留样本从原视频到最终 triplet 的完整血缘

## 2. 顶层结构

```json
{
  "video": {},
  "trace": {},
  "segments": [],
  "samples": [],
  "description_units": [],
  "temporal_structures": [],
  "text_signals": [],
  "evidence_units": [],
  "scaffold_units": [],
  "qa_drafts": [],
  "grounding_records": [],
  "qae_triplets": [],
  "revisions": [],
  "quality_records": []
}
```

## 3. 顶层字段说明

### 3.1 video
记录原视频和基础元数据。

### 3.2 trace
记录整条链路的追踪信息，如 `job_id`、`pipeline_name`、`route_family`、`training_stage`、`target_dataset_type`、`budget_profile`。

### 3.3 segments
由 A1 和 A2 共同维护，表示所有可消费片段及其轻量编排状态。

### 3.4 samples
由 A3 维护，表示代表帧和采样结果。

### 3.5 description_units
由 A4 维护，表示 frame、event、segment、context 等多粒度描述单元。

### 3.6 temporal_structures
由 A5 维护，表示时间窗、事件链、阶段关系和多跳时序线索。

### 3.7 text_signals
由 A6 维护，表示 OCR、字幕、ASR 和其他文本辅证。

### 3.8 evidence_units
由 A4/A5/A6 的规约层或 A7 共同维护，表示统一证据单元。它们可以来自：

- 时间窗
- 描述单元
- 文本信号
- 图路径或树路径
- 多源混合证据

### 3.9 scaffold_units
由 A7 维护，表示 `narration chunk`、`event tree`、`action graph`、`query scaffold` 等 canonical 骨架单元。

### 3.10 qa_drafts
由外部导入、人工标注系统或上游生成器维护，表示尚未完成 grounding 或尚未定稿的 QA 草案。

### 3.11 grounding_records
记录某条 QA 草案或某个 triplet 与证据单元之间的绑定结果，用于显式表示 post-hoc grounding 过程。

### 3.12 qae_triplets
由 A8 维护，表示最终用于出站的 `Question / Answer / Evidence` 三元组。

### 3.13 revisions
由 A9 维护，表示针对 triplet 的一致性回修记录。

### 3.14 quality_records
由 A10 维护，表示筛查分数、保留决策和拒绝原因。

## 4. 公共对象元信息
所有数组中的对象都必须包含以下公共字段：

- `id`
- `source_ids`
- `operator_name`
- `operator_version`
- `confidence`
- `status`
- `resource_refs`

状态字段推荐枚举：

- `active`
- `filtered`
- `superseded`
- `rejected`
- `accepted`

## 6. 关键对象的补充约束

### 6.1 DescriptionUnit 必须显式声明粒度
A4 不再只被看作单层粗 caption，因此 `description_units[]` 至少应显式保留：

- `granularity`
- `description_scope`
- `anchor_refs`

推荐 `granularity` 枚举：

- `frame`
- `event`
- `segment`
- `context`

### 6.2 EvidenceUnit 是统一证据锚点
不同路由不应各自维护一套 evidence 表示方式。

因此 `evidence_units[]` 至少应显式保留：

- `evidence_type`
- `ref_ids`
- `summary`
- `grounding_level`
- `anchor_refs`

推荐 `grounding_level` 枚举：

- `none`
- `weak`
- `temporal`
- `spatial`
- `spatiotemporal`
- `mixed`

### 6.3 ScaffoldUnit 必须是可复用结构
`scaffold_units[]` 不允许只是把上游字段简单拼接成一个长字符串。

每个 `scaffold_unit` 至少还应显式说明：

- `unit_type`
- `scaffold_role`
- `scaffold_summary`
- `composition_rule`
- `child_refs`
- `evidence_distribution`
- `source_evidence_unit_ids`
- `source_qa_draft_ids`

其中 `scaffold_role` 推荐枚举：

- `generation`
- `binding`
- `verification`

### 6.4 QADraft 不能直接作为训练样本出站
`qa_drafts[]` 代表尚未完成定稿的中间对象。

它们可以来自：

- 人工写题
- benchmark 导入
- narration / caption 改写后的草案
- LLM 先验生成的候选 QA

但在进入训练集之前，仍应经过 A7/A8/A9/A10 的完整链路。

### 6.5 QAETriplet 必须显式绑定证据
每条 `qae_triplet` 至少应引用以下证据之一，且默认优先引用 `evidence_unit_id`：

- `evidence_unit_id`
- `temporal_unit_id`
- `description_id`
- `text_signal_id`
- `segment_id`

如果是多跳问题，则应显式保留多个证据引用。

推荐 `triplet_origin` 枚举：

- `narration_first`
- `structure_first`
- `qa_first`

推荐 `dataset_tier` 枚举：

- `qa`
- `grounded_qa`
- `strong_grounded_qa`

## 7. 算子调用契约
每个算子统一消费如下三类输入：

- `VideoPacket`
- `OperatorConfig`
- `Selector`

### 7.1 OperatorConfig
定义算子的运行模式和参数。

推荐字段：

- `mode`
- `route_family`
- `budget_profile`
- `thresholds`
- `outputs_required`
- `retry_policy`
- `training_stage`
- `target_dataset_type`

### 7.2 Selector
定义本次算子处理哪些对象。

推荐字段：

- `segment_ids`
- `sample_ids`
- `description_ids`
- `temporal_unit_ids`
- `evidence_unit_ids`
- `scaffold_ids`
- `qa_draft_ids`
- `grounding_ids`
- `triplet_ids`
- `only_active`

## 8. 追加式写入规则
为保证链路可回溯，所有算子都要遵守以下规则：

1. 不覆写上游原始对象。
2. 需要替代旧结果时，新增对象并把旧对象状态设为 `superseded`。
3. 过滤行为必须显式写入 `filtered` 或 `rejected`。
4. 最终保留结果必须显式写入 `accepted`。

## 9. 数据流规则

### 9.1 默认主干
当前默认的 post-training 主干是：

`video -> segments -> samples -> description_units / temporal_structures / text_signals -> evidence_units -> scaffold_units -> qae_triplets -> revisions -> quality_records`

### 9.2 兼容的三条数据主线
在统一协议下，允许以下三条 ingress family：

- `narration-first`：
  `video -> text_signals / description_units -> evidence_units -> scaffold_units -> qae_triplets`
- `structure-first`：
  `video -> description_units / temporal_structures -> scaffold_units -> qae_triplets`
- `qa-first / post-hoc grounding`：
  `external qa_drafts + video evidence extraction -> evidence_units -> scaffold_units -> qae_triplets`

### 9.3 triplet 物化禁止直连原视频
A8 不允许绕过 `scaffold_units` 或规约后的 `qa_drafts[] / evidence_units[]` 直接读取原视频自由生成三元组。

### 9.4 筛查前必须回修
A10 不应直接消费 A8 的原始 triplets，而应优先消费经过 A9 回修后的版本。

## 11. 原则

- 统一协议优先于单算子优化
- 高质量 post-training 数据优先于单次推理便利
- 结构骨架优先于自由生成
- 统一证据规约优先于各路由各写一套字段
- triplet 物化优先于后验拼接
- 最终接受样本必须可回挂到结构证据
