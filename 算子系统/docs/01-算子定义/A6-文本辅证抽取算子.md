# 文本辅证抽取算子（A6）技术文档

## 1. 在系统中的定位
文本辅证抽取算子负责把视频中的 OCR、字幕、ASR 等文本信号转成可引用的辅证对象。在当前落地版中，A6 的默认角色是：

- 围绕 A8 的 query bundle 和 A4 缩小后的候选 span 做文本回看
- 为 A9 的答案回修与严格审查补充视觉盲区外的证据

它不是主视觉证据，但常常决定问题是否足够精确、答案是否足够可靠。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中偏弱`

### 2.1 文献锚点
- MME-VideoOCR
- LongVideoBench
- TimeChat-Captioner

### 2.2 从文献保留了什么
- 很多高价值问答依赖字幕、标题卡、路牌、PPT 和界面文字。
- 文本信号通常作为辅证，而不是主干算子单独出现。
- 文本结果必须结构化保存，供 grounded 回修和审查复用。

## 3. 子模式定义

### 3.1 OCR
从关键帧中识别画面文字。

### 3.2 SUBTITLE_PARSE
解析外挂字幕或容器字幕。

### 3.3 ASR_ALIGN
从语音转写中对齐片段级文本。

### 3.4 CLAIM_TARGETED_TEXT
围绕 claim 的关键词只抽取与当前 bundle 强相关的文本证据。

这是当前默认链路中的推荐模式。

## 4. 输入规范
消费 `segments[]`、`samples[]`、`query_bundles[]`，必要时读取 `subtitle_uri` 或 `asr_uri`。

### 4.1 必需输入
- `segments[].segment_id`
- `samples[].sample_id`
- `query_bundles[].bundle_id`

### 4.2 可选输入
- `video.subtitle_uri`
- `video.asr_uri`
- `temporal_events[].event_id`
- `config.text_modes`
- `config.merge_strategy`
- `config.keyword_gate`

## 5. 输出规范
写入 `text_signals[]`，推荐字段：

- `text_signal_id`
- `segment_id`
- `query_bundle_id`
- `timestamp_range`
- `text_type`
- `text`
- `language`
- `supporting_claim_ids`
- `text_role`

建议显式区分：

- `text_role=support`
- `text_role=refute`
- `text_role=context`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `text_modes` | 启用 OCR、字幕、ASR 中哪些分支 |
| `merge_strategy` | 多来源文本如何合并 |
| `language_hint` | 文本语言提示 |
| `keyword_gate` | 是否按 query 关键词过滤文本候选 |

## 7. 与上下游的绑定关系
- 上游：A3 提供采样，A8 提供 query bundle，A4 可提供候选时序范围
- 下游：A9 用文本辅证检查答案是否与视觉证据一致，或补齐视觉难以直接给出的细节

## 8. 典型失败模式
- OCR 错字导致 claim 目标偏移
- 字幕时间对齐错误，误把其他时段文本当成当前证据
- 多来源文本冲突但未显式记录置信度
- 文本抽取未被 query 约束，导致把无关语句混入最终证据摘要

## 9. 推荐使用场景
- 教学视频、网页视频、带字幕视频
- 需要 OCR / 字幕 / ASR 支撑的问题
- 需要补强视觉盲区和时间短暂提示的 grounded QA 场景
