# 文本抽取算子（A6）技术文档

## 1. 在系统中的定位
文本抽取算子负责把视频中高密度但容易被忽略的文本证据纳入统一中间态，包括画面文字、字幕和 ASR 对齐结果。它不是最终标注生成器，而是多模态证据收集器。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中偏弱`

### 2.1 文献锚点
- MME-VideoOCR
- LongVideoBench
- TimeChat-Captioner

### 2.2 从文献保留了什么
- 长视频理解高度依赖字幕和画面文字
- 纯视觉描述经常无法覆盖 PPT、标题卡、路牌、界面文字等强语义信号
- 文本证据应作为结构化输入供后续标注或裁决使用

## 3. 子模式定义

### 3.1 OCR
从关键帧中识别画面文字。

### 3.2 SUBTITLE_PARSE
解析外挂字幕或容器字幕。

### 3.3 ASR_ALIGN
从语音转写中对齐片段级文本。

### 3.4 TEXT_MERGE
合并 OCR、字幕、ASR，形成时间有序的文本证据流。

## 4. 输入规范
消费 `segments[]` 和 `samples[]`，追加 `text_signals[]`。

### 4.1 必需输入
- `segment_id`
- `sample_id` 或 `subtitle_uri/asr_uri`

### 4.2 可选输入
- `config.ocr_engine`
- `config.language`
- `config.merge_window_sec`
- `config.max_text_length`

## 5. 输出规范
每条文本证据写入 `text_signals[]`，推荐字段：

- `text_signal_id`
- `segment_id`
- `sample_id`
- `text_type`
- `timestamp_range`
- `text`
- `confidence`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `ocr_engine` | 指定 OCR 工具 |
| `language` | 文本语言 |
| `merge_window_sec` | 跨来源文本合并窗口 |
| `max_text_length` | 限制单条文本长度 |

## 7. 与上下游的绑定关系
- 上游：A3/A4 提供样本帧
- 下游：A9 直接消费文本证据生成 richer caption 或 QA
- 下游：A10 可对文本和视觉的一致性做交叉检查

## 8. 典型失败模式
- OCR 误读导致语义污染
- 字幕时间轴与片段边界不对齐
- 文本融合后信息重复或顺序错乱

## 9. 推荐使用场景
- 教程、演讲、课堂视频
- 屏幕录制、游戏 UI、软件演示
- 标题卡和字幕信息密集的网页视频
