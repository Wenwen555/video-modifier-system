# 质量筛查算子（A10）技术文档

## 1. 在系统中的定位
A10 是当前系统的最终出口。它负责对经过 A9 回修的样本做规则检查、评分、筛除和保留决策。

A10 只负责“是否出站”，不负责重新生成骨架或三元组。

在当前版本里，A10 的默认目标不是单纯筛掉坏样本，而是为 `post-training / instruction tuning` 选出更高价值、更高可靠性、grounding 更明确的 LVLM QA 数据。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- LongViTU
- Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
- Omni-Judge
- MVQA-68K

### 2.2 从文献保留了什么
- 自动构造样本必须经过过滤和评分
- 最终数据质量控制不是可选项，而是主链的一部分
- 对训练数据而言，证据质量和训练价值必须显式进入打分

## 3. 子模式定义

### 3.1 RULE_FILTER
按硬规则过滤非法样本。

### 3.2 SCORE_GATE
按多维分数决定保留与否。

### 3.3 VALUE_RANK
按样本价值排序，用于挑选高价值子集。

### 3.4 TIER_ASSIGN
根据 `grounding_level`、证据完整度和问题复杂度，将样本分配到 `qa / grounded_qa / strong_grounded_qa` 等不同数据层级。

## 4. 输入规范

### 4.1 必需输入
- `qae_triplets[].triplet_id`
- `revisions[]`

### 4.2 可选输入
- `config.rule_set`
- `config.score_threshold`
- `config.max_keep_per_video`
- `config.required_dataset_tier`
- `config.training_objective`

## 5. 输出规范
写入 `quality_records[]`，推荐字段：

- `quality_id`
- `target_triplet_id`
- `scores`
- `decision`
- `reason`
- `rank_in_video`
- `assigned_dataset_tier`

## 6. 与上下游的绑定关系
- 上游：A9
- 下游：无，A10 是默认主链路的最终出口

## 7. 典型失败模式
- 只打总分，不给出拒绝原因
- 忽视 evidence 质量，只看语言流畅性
- 高价值样本排序逻辑不透明
- 不区分普通 QA 与 strong grounded QA，导致不同强度样本混在同一训练集
- 不区分 instruction value 与 factual correctness，导致训练信号失衡

## 8. 推荐使用场景
- 自动化视频数据构造的最终出站节点
- 构造高质量子集
- 构造多难度层级的数据版本
- 为 LVLM post-training / instruction tuning 构造分层训练集
