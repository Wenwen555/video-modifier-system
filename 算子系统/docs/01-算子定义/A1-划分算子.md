# A1-划分算子

## 1. 目标
A1 是整条视频算子链的入口。它将原始视频切分为后续算子可直接消费的 `segments`，并同步产出切分结果元数据，供上下文编排、采样、结构骨架构建和回溯链路复用。

当前参考实现位于：

- `01-算子定义/impl/a1_partition_operator.py`

实现类为：

- `A1_Partition_Operator`

基类为：

- `Modifier`

这意味着它可以直接复用 `op_util.py` 中已有的 `run(dataset)` 批处理方式，同时通过 `process(instance)` 处理单条样本。

## 2. 当前实现范围
当前代码实现的是一版轻量、可直接嵌入算子系统的划分算子，重点是统一输入输出协议，而不是追求最强的视觉边界性能。

当前版本支持：

- `FIXED`
- `SEMANTIC`
- `SHOT`

当前版本不包含：

- `HYBRID`
- 多阶段全局 merge
- 跨模态联合边界判定
- ffmpeg 物化导出

## 3. Mode 对照

| Mode | 切分依据 | 必需附加输入 | 关键参数 | 输出附加分数字段 | 适用场景 |
|---|---|---|---|---|---|
| `FIXED` | 固定时间窗口 | 无 | `chunk_duration_sec` | 无 | 基线切分、低成本预处理、实时链路 |
| `SEMANTIC` | 相邻文本片段的语义变化与时间间隔 | `transcript_entries` | `similarity_threshold` `semantic_gap_sec` `min_segment_duration_sec` `max_segment_duration_sec` | `semantic_shift_score` | 教程、访谈、新闻、直播回放等文本信号较强的视频 |
| `SHOT` | 采样帧灰度直方图变化 | 可读取的视频文件 | `sensitivity_threshold` `sample_stride` `min_segment_duration_sec` | `visual_change_score` | 转场明显、镜头变化频繁的剪辑类视频 |

## 4. 算法行为

### 4.1 FIXED
`FIXED` 模式从 `0.0` 开始，以 `chunk_duration_sec` 为步长递增，直到视频结束时间。该模式只依赖视频时长信息，是当前最稳定的兜底模式。

边界类型固定为：

- `FIXED_WINDOW`

默认置信度为：

- `1.0`

### 4.2 SEMANTIC
`SEMANTIC` 模式依赖 `transcript_entries`。每条文本记录需要包含：

- `start_sec`
- `end_sec`
- `text`

实现流程如下：

1. 按 `start_sec` 对文本片段排序。
2. 使用 `TOKEN_PATTERN` 对中英文文本做轻量分词。
3. 对相邻文本片段计算 Jaccard 相似度。
4. 将 `1 - similarity` 作为 `text_shift_score`。
5. 当满足以下任一条件时，尝试在当前位置切分：

- 文本间隔 `gap_sec >= semantic_gap_sec`
- 语义变化 `text_shift_score >= similarity_threshold`
- 当前段时长达到 `max_segment_duration_sec`

只有当当前片段时长不小于 `min_segment_duration_sec` 时，切分才会生效。

边界类型固定为：

- `SEMANTIC_SHIFT`

默认置信度为：

- `0.85`

输出附加字段为：

- `semantic_shift_score`

### 4.3 SHOT
`SHOT` 模式使用 OpenCV 读取视频帧，并按 `sample_stride` 做稀疏采样。对每个采样帧：

1. 转灰度图。
2. 计算 64-bin 直方图。
3. 与上一采样帧做 Bhattacharyya 距离比较。
4. 若差异值达到 `sensitivity_threshold`，且与上一个边界的时间距离不小于 `min_segment_duration_sec`，则产生新的 cut。

边界类型固定为：

- `HARD_CUT`

默认置信度为：

- `0.90`

输出附加字段为：

- `visual_change_score`

依赖要求：

- `opencv-python`

## 5. 输入协议
A1 的 `process(instance)` 接受单条字典样本。当前实现从样本中读取视频信息和可选文本信息，并返回附加了切分结果的新样本。

### 5.1 必需字段

| 字段 | 说明 |
|---|---|
| `video_path` 或 `video_uri` | 视频文件路径。`video_path` 来自 `Modifier.video_key`，也兼容直接传入 `video_uri`。 |

### 5.2 推荐字段

| 字段 | 说明 |
|---|---|
| `video_id` | 视频唯一标识。缺失时自动由文件名生成。 |
| `video_metadata.fps` 或 `fps` | 帧率。 |
| `video_metadata.total_frames` 或 `total_frames` | 总帧数。 |
| `video_metadata.duration_sec` 或 `duration_sec` | 视频总时长。 |
| `transcript_entries` | 文本切分输入，仅 `SEMANTIC` 模式需要。 |

### 5.3 元数据补全逻辑
如果 `fps` 缺失，或者 `total_frames` 与 `duration_sec` 两者都缺失，算子会尝试使用 OpenCV 自动探测视频元数据。

如果探测失败，会直接报错。

## 6. 输出协议
`process(instance)` 会在原记录上追加以下字段：

| 字段 | 说明 |
|---|---|
| `video_id` | 补全后的视频 ID。 |
| `video_uri` | 归一化后的输入视频路径。 |
| `video_metadata` | 标准化后的视频元数据。 |
| `segments` | 最终切分结果列表。 |
| `partition_result` | 完整算子输出，包括全局指标。 |
| `partition_mode` | 实际执行的切分模式。 |
| `partition_metrics` | 当前仅包含 `total_segments` 和 `processing_time_ms`。 |

每个 `segment` 至少包含：

| 字段 | 说明 |
|---|---|
| `id` / `segment_id` | 片段 ID，格式为 `seg_0001`。 |
| `source_ids` | 来源视频 ID 列表。 |
| `timestamp_range.start_sec` | 起始时间。 |
| `timestamp_range.end_sec` | 结束时间。 |
| `frame_range.start_frame` | 起始帧。 |
| `frame_range.end_frame` | 结束帧。 |
| `duration_sec` | 片段时长。 |
| `partition_mode` | 生成该片段的 mode。 |
| `boundary_type` | 边界类型。首段固定为 `START`。 |
| `confidence` | 片段置信度。 |
| `status` | 当前固定为 `active`。 |
| `operator_name` | 当前算子名。 |
| `operator_version` | 当前实现版本。 |
| `resource_refs` | 资源引用字典，默认空字典。 |

按 mode 不同，片段还会追加：

- `semantic_shift_score`
- `visual_change_score`

## 7. 参数说明

| 参数 | 默认值 | 作用 |
|---|---:|---|
| `partition_mode` | `FIXED` | 选择切分模式。 |
| `chunk_duration_sec` | `5.0` | `FIXED` 模式下的时间窗口。 |
| `sensitivity_threshold` | `0.45` | `SHOT` 模式的镜头变化阈值。 |
| `similarity_threshold` | `0.70` | `SEMANTIC` 模式的语义变化阈值。 |
| `min_segment_duration_sec` | `2.0` | 最短片段时长。低于该值的片段可被后处理合并。 |
| `max_segment_duration_sec` | `None` | `SEMANTIC` 模式下的最长片段约束。 |
| `merge_adjacent_small_segments` | `True` | 是否合并过短相邻片段。 |
| `sample_stride` | `6` | `SHOT` 模式的视频采样步长。 |
| `semantic_gap_sec` | `3.0` | `SEMANTIC` 模式下的文本时间间隔阈值。 |
| `strict` | `True` | 异常是否直接抛出。 |

## 8. 后处理规则
当前实现内置了一层轻量 verify 逻辑，核心是边界归一化和小片段合并。

### 8.1 边界归一化
算子会对边界做如下处理：

- 限制在 `[0, duration_sec]` 范围内
- 去重并排序
- 确保起点包含 `0.0`
- 确保终点包含视频结束时间

### 8.2 小片段合并
当 `merge_adjacent_small_segments=True` 时：

- 若某个片段时长小于 `min_segment_duration_sec`
- 则并入前一个片段
- 合并后边界类型改为 `MERGED_SMALL_SEGMENT`
- 合并后置信度取前后片段均值

最后一个片段的结束时间会被强制对齐到视频结束时间。

## 9. 错误处理
当前实现支持两种行为：

| `strict` | 行为 |
|---|---|
| `True` | 直接抛出异常，适合离线构建或调试。 |
| `False` | 返回空 `segments`，并在 `partition_result` 与 `partition_error` 中写入错误信息。 |

常见报错来源包括：

- 缺少 `video_path` 或 `video_uri`
- 缺少视频元数据且无法自动探测
- `SEMANTIC` 模式下未提供 `transcript_entries`
- `SHOT` 模式下缺少 OpenCV
- 视频文件无法打开

## 10. 与 `op_util.py` 的对齐方式
当前实现遵循 `Modifier` 的使用习惯，适合直接接入现有算子系统。

对齐点如下：

- 通过 `process(instance)` 处理单条样本
- 通过继承的 `run(dataset)` 处理数据集
- 通过 `operator_name` 与 `accelerator` 继承统一算子元信息
- 输出记录保留原字段，并追加标准化的划分结果字段

最小调用示例：

```python
from a1_partition_operator import A1_Partition_Operator

operator = A1_Partition_Operator(
    partition_mode="SEMANTIC",
    similarity_threshold=0.70,
    semantic_gap_sec=3.0,
)

sample = {
    "video_path": "demo.mp4",
    "video_id": "demo_001",
    "fps": 30,
    "duration_sec": 60.0,
    "transcript_entries": [
        {"start_sec": 0.0, "end_sec": 8.0, "text": "今天我们先介绍数据预处理。"},
        {"start_sec": 8.5, "end_sec": 18.0, "text": "下面进入模型训练部分。"},
    ],
}

result = operator.process(sample)
segments = result["segments"]
```

## 12. 当前实现边界
这份实现已经能作为系统内 A1 的统一切分入口，但仍然属于轻量参考实现。后续如果继续演进，优先补强的方向应当是：

- 引入质量检测和性能更好（语义连续的划分）模块
- 将 `SHOT` 从直方图差异升级为更稳的边界检测器
- 将 `SEMANTIC` 从文本 Jaccard 升级为 embedding 级相似度
- 在输出中增加更细的边界评分、证据和 profile 标识
