# 候选选择与质量裁决算子（A10）技术文档

## 1. 在系统中的定位
候选选择与质量裁决算子是整条链路的最终出口。它负责从 A9 生成的候选标注中筛出高质量样本，并显式记录保留或拒绝的原因。

它的重点不只是“打分”，而是“选择 + 解释 + 最终决策”。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- LongViTU
- VDC-Agent
- MVQA-68K
- Omni-Judge

### 2.2 从文献保留了什么
- 自动生成标注必须经过后验筛选
- 评分应是多维度的，而不是单个总分
- 最终需要有可解释的保留和拒绝理由

## 3. 子模式定义

### 3.1 RULE_CHECK
使用规则检查明显错误，如证据缺失、时间越界、对象引用缺失。

### 3.2 VLM_JUDGE
利用多模态模型对候选标注做质量评判。

### 3.3 CONSISTENCY_SCORE
检查文本、时序和空间证据之间的一致性。

### 3.4 FILTER_GATE
根据综合得分决定保留、拒绝或回退重试。

## 4. 输入规范
消费 `annotations[]` 及其关联证据。

### 4.1 必需输入
- `annotations[].annotation_id`
- `annotations[].evidence_refs`

### 4.2 可选输入
- `config.rule_set`
- `config.score_threshold`
- `config.retry_target`
- `config.max_candidates_keep`

## 5. 输出规范
写入 `quality[]`，并更新 `annotations[].status`。

质量对象推荐字段：

- `quality_id`
- `target_annotation_id`
- `scores.factuality`
- `scores.temporal_consistency`
- `scores.spatial_grounding`
- `scores.text_visual_alignment`
- `decision`
- `reason`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `score_threshold` | 最终保留阈值 |
| `max_candidates_keep` | 每个目标保留候选数 |
| `rule_set` | 规则集选择 |
| `retry_target` | 被拒样本的回退路径 |

## 7. 与上下游的绑定关系
- 上游：A9 提供候选标注
- 下游：无视频算子下游，A10 是系统出口

## 8. 典型失败模式
- 评分偏向语言流畅性而忽略视觉真实性
- 规则过严导致召回过低
- 多维评分存在冲突但没有明确决策逻辑

## 9. 推荐使用场景
- 自动化数据构建的最终过滤
- 高质量数据子集筛选
- 汇报和实验中展示“从候选到黄金样本”的流程
