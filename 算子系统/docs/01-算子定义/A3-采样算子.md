# A3-采样算子

## 1. 定位
A3 现在是系统内唯一的采样入口，统一承担原 A3 的基础采样和原 A4 的自适应采样职责。

合并后的原因很直接：

- 两者消费的都是 `segments[]`
- 两者产出的都是 `samples[]`
- 差别主要在于“预算怎么分配”，而不是“是否进入下一个独立阶段”

因此，系统不再把“基础采样”和“自适应采样”拆成两个连续算子，而是把它们收敛为同一算子的不同模式。

当前参考实现位于：

- `01-算子定义/impl/a3_sampling_operator.py`

当前代码已经实现的模式是基础采样部分；自适应模式目前作为系统级接口保留位写入文档，便于后续继续扩展。

## 2. 模式分层

| 层级 | Mode | 核心思路 | 关键参数 | 适用场景 |
|---|---|---|---|---|
| 基础 | `CENTER` | 每段取中心代表帧 | 无 | 最低成本概览、稳定基线 |
| 基础 | `UNIFORM` | 每段均匀抽帧 | `num_frames` `target_fps` `max_frames_per_segment` | 长片段概览、多帧基线 |
| 基础 | `HEAD_TAIL` | 抓首尾两端状态 | `max_frames_per_segment` | 前后状态对比 |
| 基础 | `STATE` | 在首尾内缩采样，避开切点噪声 | `padding_frames` `max_frames_per_segment` | 状态变化、切点不稳定场景 |
| 自适应 | `KEYFRAME` | 基于代表性和冗余压缩选择关键帧 | `budget_frames` `redundancy_threshold` | 长视频关键帧保留 |
| 自适应 | `FEATURE_DRIFT` | 在语义漂移处增加采样密度 | `budget_frames` `semantic_drift_threshold` | 内容变化不均匀的视频 |
| 自适应 | `MOTION_AWARE` | 在运动强、轨迹变动大的位置追加预算 | `budget_frames` `motion_threshold` | 动作密集、局部动态明显的视频 |

## 3. 输入与输出

### 3.1 输入
必需字段：

- `video_path` 或 `video_uri`
- `segments`
- `segments[].frame_range.start_frame`
- `segments[].frame_range.end_frame`

推荐字段：

- `video_id`
- `video_metadata.fps` 或 `fps`
- `video_metadata.total_frames` 或 `total_frames`
- `video_metadata.duration_sec` 或 `duration_sec`
- `segments[].segment_id`

当前实现会对片段做统一归一化：

- 约束 `start_frame >= 0`
- 约束 `end_frame <= total_frames`
- 保证每段至少可采 1 帧
- 缺失 `segment_id` 时自动补齐

### 3.2 输出
算子在原记录上追加：

- `samples`
- `sampling_result`
- `sampling_mode`
- `sampling_metrics`

每个 `sample` 至少包含：

- `sample_id`
- `segment_id`
- `timestamp_sec`
- `frame_index`
- `sample_type`
- `sampling_mode`
- `relative_position`
- `source_ids`
- `confidence`
- `status`

自适应模式建议额外写入：

- `sample_reason`
- `information_score`
- `budget_source`

## 4. 关键参数

| 参数 | 默认值 | 作用 |
|---|---:|---|
| `sampling_mode` | `CENTER` | 统一指定基础或自适应模式 |
| `num_frames` | `4` | `UNIFORM` 默认采样帧数 |
| `target_fps` | `None` | 按目标帧率决定采样密度 |
| `padding_frames` | `2` | `STATE` 模式的首尾偏移 |
| `max_frames_per_segment` | `None` | 单片段样本硬上限 |
| `budget_frames` | `None` | 自适应模式的预算总量 |
| `redundancy_threshold` | `None` | `KEYFRAME` 的去冗余阈值 |
| `semantic_drift_threshold` | `None` | `FEATURE_DRIFT` 的触发阈值 |
| `motion_threshold` | `None` | `MOTION_AWARE` 的触发阈值 |
| `strict` | `True` | 出错时是否直接抛异常 |

## 5. 与上下游的关系

- 上游：A1 提供片段，A2 过滤明显无效片段
- 下游：A4 对样本进一步去重压缩
- 下游：A5、A6、A7 基于样本提取文本、空间和时序证据
- 下游：A8、A9 在采样结果及其证据上完成标注生成与质量裁决

需要强调的是，A3 内部允许“先粗采样，再自适应重分配预算”，但这已经属于同一算子内部策略，不再额外拆出 A4。

## 6. 当前实现边界

当前 `impl/a3_sampling_operator.py` 已实现：

- `CENTER`
- `UNIFORM`
- `HEAD_TAIL`
- `STATE`

当前尚未在参考代码中落地，但已在系统设计中保留接口的能力包括：

- `KEYFRAME`
- `FEATURE_DRIFT`
- `MOTION_AWARE`
- 图像质量过滤
- 多模态重排和二次预算分配

因此，A3 的现阶段定位是：协议已经统一，基础模式可运行，自适应模式可继续往里长，而不再需要新增一个并列采样算子。
