# A3-采样算子

## 1. 定位
A3 现在是系统内唯一的采样入口，统一承担基础采样和自适应采样职责。

合并后的原因很直接：

- 两者消费的都是 `segments[]`
- 两者产出的都是 `samples[]`
- 差别主要在于“预算怎么分配”，而不是“是否进入下一个独立阶段”

因此，系统不再把“基础采样”和“自适应采样”拆成两个连续算子，而是把它们收敛为同一算子的不同模式。

当前参考实现位于：

- `01-算子定义/impl/a3_sampling_operator.py`

当前代码已经实现的模式是基础采样部分；自适应模式目前作为系统级接口保留位写入文档，便于后续继续扩展。

## 2. 模式分层

| 层级 | Mode | 核心思路 | 关键参数 | 适用场景 |
|---|---|---|---|---|
| 基础 | `CENTER` | 每段取中心代表帧 | 无 | 最低成本概览、稳定基线 |
| 基础 | `UNIFORM` | 每段均匀抽帧 | `num_frames` `target_fps` `max_frames_per_segment` | 长片段概览、多帧基线 |
| 基础 | `HEAD_TAIL` | 抓首尾两端状态 | `max_frames_per_segment` | 前后状态对比 |
| 基础 | `STATE` | 在首尾内缩采样，避开切点噪声 | `padding_frames` `max_frames_per_segment` | 状态变化、切点不稳定场景 |
| 自适应 | `SEMANTIC-AWARE` | 提高采样多样性 | `budget_frames` `semantic_drift_threshold` | 内容变化不均匀的视频 |
| 自适应 | `MOTION_AWARE` | 在运动强、轨迹变动大的位置追加预算 | `budget_frames` `motion_threshold` | 动作密集、局部动态明显的视频 |

## 3. 输入与输出

### 3.1 输入
必需字段：

- `video_path` 或 `video_uri`
- `segments`
- `segments[].frame_range.start_frame`
- `segments[].frame_range.end_frame`

推荐字段：

- `video_id`
- `video_metadata.fps` 或 `fps`
- `video_metadata.total_frames` 或 `total_frames`
- `video_metadata.duration_sec` 或 `duration_sec`
- `segments[].segment_id`

当前实现会对片段做统一归一化：

- 约束 `start_frame >= 0`
- 约束 `end_frame <= total_frames`
- 保证每段至少可采 1 帧
- 缺失 `segment_id` 时自动补齐

### 3.2 输出
算子在原记录上追加：

- `samples`
- `sampling_result`
- `sampling_mode`
- `sampling_metrics`

每个 `sample` 至少包含：

- `sample_id`
- `segment_id`
- `timestamp_sec`
- `frame_index`
- `sample_type`
- `sampling_mode`
- `relative_position`
- `source_ids`
- `confidence`
- `status`

自适应模式建议额外写入：

- `sample_reason`
- `information_score`
- `budget_source`

## 4. 关键参数

| 参数 | 默认值 | 作用 |
|---|---:|---|
| `sampling_mode` | `CENTER` | 统一指定基础或自适应模式 |
| `num_frames` | `4` | `UNIFORM` 默认采样帧数 |
| `target_fps` | `None` | 按目标帧率决定采样密度 |
| `padding_frames` | `2` | `STATE` 模式的首尾偏移 |
| `max_frames_per_segment` | `None` | 单片段样本硬上限 |
| `semantic_min_distance_threshold` | `None` | `SEMANTIC_AWARE` 的触发阈值 |
| `motion_threshold` | `None` | `MOTION_AWARE` 的触发阈值 |
| `strict` | `True` | 出错时是否直接抛异常 |

## 5. 与上下游的关系

- 上游：A1 提供原始片段，A2 对片段做轻量清洗和简单编排
- 默认下游：A4、A5、A6 分别在采样结果上生成多粒度描述、时序结构和文本辅证
- 骨架下游：A7 在这些结构化产物上构建 scaffold
- 终端下游：A8、A9、A10 基于骨架完成 Q/A/E 共同合成、回修和筛查

需要强调的是，A3 内部允许“先粗采样，再自适应重分配预算”，但这已经属于同一算子内部策略，不再额外拆出并列的自适应采样算子。当前默认路线中，A3 的采样结果直接服务于结构骨架提取与 Q/A/E 共同合成。

## 6. 当前实现边界

当前 `impl/a3_sampling_operator.py` 已实现：

- `CENTER`
- `UNIFORM`
- `HEAD_TAIL`
- `STATE`

因此，A3 的现阶段定位是：协议已经统一，基础模式可运行，自适应模式可继续往里长，而不再需要新增一个并列采样算子。
