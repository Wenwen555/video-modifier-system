# 时序定位与变化摘要算子（A7）技术文档

## 1. 在系统中的定位
本算子负责把视频片段内部的“什么时候发生了什么变化”显式写出来。它不是只做简单摘要，而是同时承担时序定位和变化建模两种职责。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`很强`

### 2.1 文献锚点
- LeAdQA
- ED-VTG
- Video-in-the-Loop
- TimeChat-Captioner
- Think with Grounding

### 2.2 从文献保留了什么
- 关键时刻需要显式时间边界
- 长视频理解需要区分全局内容和关键瞬间
- 带时间戳的结构化摘要比单句描述更有训练价值

## 3. 子模式定义

### 3.1 HEAD_TAIL_DIFF
基于首尾帧差异概括片段变化。

### 3.2 TRACK_SUMMARY
利用对象或局部区域随时间的变化生成摘要。

### 3.3 STATE_CHANGE
显式识别状态突变时刻。

### 3.4 TEMPORAL_EVENT
输出若干带时间边界的局部事件描述。

## 4. 输入规范
消费 `segments[]`、`samples[]`，可选消费 `regions[]` 和 `text_signals[]`。

### 4.1 必需输入
- `segment_id`
- `sample_ids`
- `config.mode`

### 4.2 可选输入
- `region_ids`
- `text_signal_ids`
- `config.max_events_per_segment`
- `config.min_event_duration_sec`

## 5. 输出规范
写入 `temporal_events[]`，推荐字段：

- `event_id`
- `segment_id`
- `start_sec`
- `end_sec`
- `event_type`
- `summary`
- `evidence_refs`
- `confidence`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `max_events_per_segment` | 限制事件数量 |
| `min_event_duration_sec` | 过滤极短事件 |
| `state_change_threshold` | 状态变化触发阈值 |
| `use_text_signals` | 是否融合文本证据 |

## 7. 与上下游的绑定关系
- 上游：A3/A4/A5/A6 提供视觉、文本和局部区域证据
- 下游：A8 用它生成 temporal caption 和时序 QA
- 下游：A9 用它检验标注是否与时间边界一致

## 8. 典型失败模式
- 事件边界太粗
- 时间摘要过于抽象，缺乏关键对象
- 局部事件太多，反而破坏简洁性

## 9. 推荐使用场景
- 长视频关键时刻定位
- 动作转折明显的视频
- 强调时间敏感问答的数据构建
