# 问题规划算子（A7）技术文档

## 1. 在系统中的定位
问题规划算子负责把时序、空间和文本三类证据组织成“可以问什么、应该怎么问、答案应该落在哪类证据上”的问题蓝图。

它是从证据走向 QA 的桥梁，而不是直接负责文本生成。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- LongViTU
- VDC-Agent
- VideoRefer Suite
- Think with Grounding

### 2.2 从文献保留了什么
- 高质量 QA 不是直接从视频端到端吐出来的。
- 需要先决定问题目标、证据引用、答案形式和难度。
- 规划阶段往往以内嵌 agent 或 prompt decomposition 的形式出现在论文中。

## 3. 子模式定义

### 3.1 FACTUAL_PLAN
规划事实类问题。

### 3.2 TEMPORAL_PLAN
规划基于关键时刻或事件顺序的问题。

### 3.3 SPATIAL_PLAN
规划基于对象、区域或局部变化的问题。

### 3.4 MULTIHOP_PLAN
规划需要跨多类证据联合回答的问题。

## 4. 输入规范
消费 `temporal_events[]`、`regions[]`、`text_signals[]`，并可回看 `segments[]` 与 `samples[]`。

### 4.1 必需输入
- 至少一种证据分支结果
- `config.plan_mode`

### 4.2 可选输入
- `config.target_difficulty`
- `config.max_questions_per_target`
- `config.answer_type_set`
- `config.require_multi_evidence`

## 5. 输出规范
写入 `question_plans[]`，推荐字段：

- `plan_id`
- `question_intent`
- `target_ids`
- `evidence_refs`
- `answer_type`
- `difficulty`
- `prompt_blueprint`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `target_difficulty` | 控制题目难度 |
| `max_questions_per_target` | 每个目标生成多少问题计划 |
| `answer_type_set` | 控制答案类型，如 span、entity、yes/no |
| `require_multi_evidence` | 是否强制跨证据规划 |

## 7. 与上下游的绑定关系
- 上游：A4、A5、A6
- 下游：A8 依据问题计划生成 QA，A9 可检查计划与最终 QA 是否一致

## 8. 典型失败模式
- 题目看起来复杂，但证据锚点不清楚
- 同一证据被重复规划成低价值近义问题
- 难度与可回答性失衡

## 9. 推荐使用场景
- 需要规模化构建 QA 数据
- 需要控制题型分布和难度分布
- 需要展示“证据到问题”的中间推导链
