# 多粒度描述算子（A4）技术文档

## 1. 在系统中的定位
A4 负责在 `A2 + A3` 给出的轻量编排结果和采样结果上生成可复用的描述单元。

它不是最终标注，也不应被狭义理解成“只做一层粗 caption”。当前链路里，A4 的真正定位是：

- 为 frame-level、event-level、segment-level 乃至 context-level 提供描述底座
- 为 A5 时序结构抽取提供语义标签
- 为 A7 结构骨架构建提供节点内容
- 为 A8 Q/A/E 共同合成提供可问的语义素材

因此，A4 更适合被理解为“多粒度描述层”，而不是单次“粗 caption”步骤。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中`

### 2.1 文献锚点
- TimeChat-Captioner
- Shot2Story
- LongViTU

### 2.2 从文献保留了什么
- 长视频数据构造通常需要多粒度描述，而不是只保留一层 caption
- 在 LongViTU 这类链路里，frame-level、event-level、segment-level 描述会反复出现
- 这些描述是 tree、graph、chunk 和后续问答生成的基础素材

## 3. 子模式定义

### 3.1 FRAME_DESCRIPTION
围绕关键帧生成 frame-level 描述。

### 3.2 EVENT_DESCRIPTION
围绕事件窗生成 event-level 描述。

### 3.3 SEGMENT_DESCRIPTION
围绕 segment 生成 segment-level 描述。

### 3.4 CONTEXT_DESCRIPTION
结合 A2 的 `stitch_group` 或固定 `context window` 生成 context-level 描述。

### 3.5 MULTI_SENTENCE_SUMMARY
输出多句描述，为后续骨架构建保留更丰富语义。

## 4. 输入规范

### 4.1 必需输入
- `segments[].segment_id`
- `samples[].sample_id`

### 4.2 可选输入
- `segments[].stitch_group_id`
- `segments[].context_window_refs`
- `temporal_structures[]`
- `config.description_granularity`
- `config.max_sentences`

## 5. 输出规范
写入 `description_units[]`，推荐字段：

- `description_id`
- `segment_id`
- `description_text`
- `description_scope`
- `granularity`
- `anchor_refs`
- `summary_tags`

其中：

- `granularity` 推荐取值：`frame` / `event` / `segment` / `context`
- `anchor_refs` 可引用 `sample_id`、`temporal_unit_id` 或 `segment_id`

## 6. 与上下游的绑定关系
- 上游：A2、A3
- 下游：A5、A7、A8

## 7. 典型失败模式
- 把多粒度描述都压成一句泛化文本，导致后续结构抽取失真
- 只写单帧内容，丢失片段级和事件级动态语义
- 同一上下文单元在不同粒度下生成彼此冲突的描述
- 把 event-level 和 segment-level 描述混为一谈，导致骨架节点粒度不稳

## 8. 推荐使用场景
- 结构骨架式数据构造默认开启
- 需要从 frame/event/segment 多个层级派生后续样本时
- 需要构建 tree、graph、chunk 等可复用骨架时
