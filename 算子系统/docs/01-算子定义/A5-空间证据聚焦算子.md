# 空间证据聚焦算子（A5）技术文档

## 1. 在系统中的定位
空间证据聚焦算子负责把“看整帧”转化为“看局部对象、区域或标记集合”。在当前落地版中，A5 的默认角色是：

- 在 A4 已缩小的候选 span 内继续做局部视觉取证
- 为 A9 的答案回修与严格审查提供对象级、区域级证据

因此，A5 在默认主链路中不再被表述为“无条件先行的 region proposal”，而是一个**被 query bundle 和 temporal span 共同约束**的回看节点。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`很强`

### 2.1 文献锚点
- VideoRefer Suite
- SpaceVLLM
- CAT-V
- VideoLoom
- SlotVTG

### 2.2 从文献保留了什么
- 对象级和区域级证据应被显式建模。
- 空间 grounding 不只是可视化效果，而是问答证据的一部分。
- 很多工作虽然看起来像 prompt-based region proposal，但本质上依然是为下游任务或问答服务的 task-conditioned 取证。

## 3. 子模式定义

### 3.1 SPAN_CONDITIONED_REGION
在 A4 已缩小的候选 span 内，围绕 query bundle 的对象线索做区域提取。

这是当前默认模式。

### 3.2 OBJECT_TRACK_FOCUS
围绕关键对象跨帧聚焦，适合需要看对象状态变化的场景。

### 3.3 SOM_TAG
给关键对象或区域打稳定标记，方便下游复核或可视化。

### 3.4 LIGHT_PROPOSAL
只做轻量显著区域提取，作为低成本辅助模式。

## 4. 输入规范
消费 `samples[]`、`temporal_events[]`、`query_bundles[]`。

### 4.1 必需输入
- `samples[].sample_id`
- `query_bundles[].bundle_id`
- `temporal_events[].event_id`

### 4.2 可选输入
- `samples[].resource_refs.frame_uri`
- `config.region_mode`
- `config.max_regions_per_event`
- `config.target_hint`
- `draft_units[].claim_slots`

## 5. 输出规范
写入 `regions[]`，推荐字段：

- `region_id`
- `sample_id`
- `query_bundle_id`
- `event_id`
- `region_type`
- `bbox` 或 `mask_uri`
- `object_tag`
- `supporting_claim_ids`
- `region_role`
- `confidence`

其中建议显式区分：

- `region_role=support`
- `region_role=refute`
- `region_role=context`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `region_mode` | 选择框、mask 或 SoM 标记 |
| `max_regions_per_event` | 每个时序事件最多保留区域数 |
| `target_hint` | 指定关注对象或区域类别 |
| `track_across_frames` | 是否启用跨帧对象跟踪 |

## 7. 与上下游的绑定关系
- 上游：A4 提供候选 span，A8 提供 query bundle
- 上游补充：A3 提供底层采样，A7 的 claim slots 可补充对象提示
- 下游：A9 直接消费区域证据，决定是否保留、回修或拒绝草案中的相关 claim

## 8. 典型失败模式
- 区域过多，导致真正关键对象被稀释
- 没有先做 A4 缩 span，导致 A5 的成本快速膨胀
- 只检测显著物体，漏掉真正与答案相关的局部线索
- 跨帧对象标签不稳定，导致 claim 对齐失败

## 9. 推荐使用场景
- 对象中心问答
- 时空联合 grounding
- 需要局部视觉证据支撑最终回修与审查的场景
