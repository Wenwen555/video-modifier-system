# 空间证据聚焦算子（A5）技术文档

## 1. 在系统中的定位
空间证据聚焦算子负责把“看整帧”转化为“看局部对象、区域或标记集合”。它是从全局视频理解走向对象级 grounding 的核心入口。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`很强`

### 2.1 文献锚点
- VideoRefer Suite
- SpaceVLLM
- CAT-V
- VideoLoom
- SlotVTG

### 2.2 从文献保留了什么
- 对象级和区域级证据应被显式建模。
- 空间 grounding 不只是可视化效果，而是问答证据的一部分。
- 区域证据需要可被下游问题规划与核验直接引用。

## 3. 子模式定义

### 3.1 BBOX
输出显著对象或目标框。

### 3.2 MASK
输出实例级或区域级 mask。

### 3.3 SOM_TAG
给关键对象或区域打上稳定标记。

## 4. 输入规范
消费 `samples[]`，必要时可结合 `temporal_events[]` 做时序条件聚焦。

### 4.1 必需输入
- `samples[].sample_id`
- `samples[].resource_refs.frame_uri`

### 4.2 可选输入
- `config.region_mode`
- `config.max_regions_per_sample`
- `config.target_hint`

## 5. 输出规范
写入 `regions[]`，推荐字段：

- `region_id`
- `sample_id`
- `region_type`
- `bbox` 或 `mask_uri`
- `object_tag`
- `confidence`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `region_mode` | 选择框、mask 或 SoM 标记 |
| `max_regions_per_sample` | 每帧最多保留区域数 |
| `target_hint` | 指定关注对象或区域类别 |

## 7. 与上下游的绑定关系
- 上游：A3 采样算子
- 下游：A7 用空间证据规划问题，A8/A9 可直接引用区域证据

## 8. 典型失败模式
- 区域过多，导致问题目标不清晰
- 只检测显著物体，漏掉真正与答案相关的局部线索
- 跨帧对象标签不稳定

## 9. 推荐使用场景
- 对象中心问答
- 场景复杂、局部信息密集的视频
- 需要展示 grounding 证据链的场景
