# 结构骨架构建算子（A7）技术文档

## 1. 在系统中的定位
A7 是当前系统的数据收敛中枢。它负责把 A4、A5、A6 的产物，以及必要时的 `qa_drafts[]`，组织成可复用的结构骨架。

在当前版本里，A7 不再只服务 `structure-first`。它同时负责让以下三条数据主线收敛到统一中间层：

- `narration / caption-first`
- `structure-first`
- `QA-first / post-hoc grounding`

结构骨架可以是：

- narration chunk
- hierarchical pack
- event tree
- action graph
- chunk graph
- query scaffold
- verification scaffold
- 其他可共同合成的结构单元

A7 的目标不是直接生成问答，而是回答：

- 哪些描述单元、时间窗和文本信号应该属于同一个单元
- 哪些证据单元应该围绕某条 QA 草案被组织起来
- 这些单元之间是什么关系
- 后续 A8 应该在哪个粒度上生成或定稿 `Q/A/E`

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- Grounded Question-Answering in Long Egocentric Videos
- Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
- LongViTU

### 2.2 从文献保留了什么
- 数据构造往往先有 chunk、tree 或 graph
- 这些结构决定后续问什么、答什么、引用哪些证据
- 对于 QA-first 路线，问题本身也需要被纳入统一 scaffold，而不是绕开结构层直接做补证

## 3. 子模式定义

### 3.1 CHUNK_PACK
把相邻描述和时间窗组织成 chunk。

### 3.2 HIERARCHICAL_PACK
在已有描述和时序结构上构造 coarse / fine 共存的 pack。

这部分职责在当前版本中被明确归入 A7，而不再放在 A2。原因是 pack 的形成通常依赖：

- 多粒度描述
- 事件或阶段关系
- 显式的层级组织规则

这已经属于骨架构建，而不是轻量前处理。

### 3.3 TREE_BUILD
构造层级树。

### 3.4 GRAPH_BUILD
构造事件图、动作图或多跳关系图。

### 3.5 QUERY_SCAFFOLD
围绕某条 `qa_draft` 组织候选证据，构造 query-conditioned scaffold。

它的目标不是回答问题，而是把：

- 问题草案
- 候选时间窗
- 描述单元
- 文本辅证

绑定成后续 A8 可消费的统一对象。

## 4. 输入规范

### 4.1 必需输入
以下两类输入至少满足一类：

- 结构构件类：`description_units[]`、`temporal_structures[]`
- 绑定构件类：`qa_drafts[]`、`evidence_units[]`

### 4.2 可选输入
- `text_signals[]`
- `evidence_units[]`
- `qa_drafts[]`
- `config.unit_type`
- `config.max_children_per_unit`
- `config.allow_cross_segment_link`
- `config.scaffold_role`

## 5. 输出规范
写入 `scaffold_units[]`，推荐字段：

- `scaffold_id`
- `unit_type`
- `scaffold_role`
- `source_description_ids`
- `source_temporal_unit_ids`
- `source_text_signal_ids`
- `source_evidence_unit_ids`
- `source_qa_draft_ids`
- `scaffold_summary`
- `composition_rule`
- `child_refs`
- `evidence_distribution`

### 5.1 `hierarchical pack` 示例
原先放在 A2 的 pack 示例，现在作为 A7 的 `hierarchical_pack` scaffold 变体保留。字段命名对齐到当前 `scaffold_units[]` 协议：

```json
{
  "scaffold_id": "pack_03",
  "unit_type": "hierarchical_pack",
  "scaffold_role": "generation",
  "scaffold_summary": "课程导入与第一节开始",
  "timestamp_range": {
    "start_sec": 120.0,
    "end_sec": 168.0
  },
  "frame_range": {
    "start_frame": 3600,
    "end_frame": 5040
  },
  "child_refs": [
    "seg_10",
    "seg_11",
    "seg_12",
    "seg_13"
  ],
  "anchor_segment_ids": [
    "seg_12",
    "seg_13"
  ],
  "hierarchy_level": "coarse",
  "composition_rule": "same_topic_progression",
  "status": "active"
}
```

## 6. 与上下游的绑定关系
- 上游：A4、A5、A6
- 兼容上游：`qa_drafts[]`、`evidence_units[]`
- 下游：A8、A9、A10

## 7. 典型失败模式
- 骨架单元过大，导致问题粒度太粗
- 骨架单元过碎，导致 evidence 复用性变差
- tree / graph 关系只存文本，不存显式 child refs
- query scaffold 只挂问题，不挂候选证据
- narration scaffold 没有显式时间锚点，导致后续 grounding 强度不足

## 8. 推荐使用场景
- 默认数据构造主链路
- narration / caption-first 数据构造
- QA-first / post-hoc grounding 的证据绑定入口
- 需要规模化生成可复用 Q/A/E 样本时
