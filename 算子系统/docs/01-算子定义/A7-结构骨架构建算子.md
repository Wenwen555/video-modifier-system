# 结构骨架构建算子（A7）技术文档

## 1. 在系统中的定位
A7 是当前系统的数据收敛中枢。它负责把 A4、A5、A6 的产物，以及必要时的 `qa_drafts[]`，组织成可复用的结构骨架。

在当前版本里，A7 不再只服务 `structure-first`。它同时负责让以下三条数据主线收敛到统一中间层：

- `narration / caption-first`
- `structure-first`
- `QA-first / post-hoc grounding`

在初步实现中，当前 A7 只考虑 4 类代表性的 canonical scaffold：

- `narration chunk`
- `event tree`
- `action graph`
- `query scaffold`


A7 的目标不是直接生成问答，而是回答：

- 哪些描述单元、时间窗和文本信号应该属于同一个单元
- 哪些证据单元应该围绕某条 QA 草案被组织起来
- 这些单元之间是什么关系
- 后续 A8 应该在哪个粒度上生成或定稿 `Q/A/E`

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- Grounded Question-Answering in Long Egocentric Videos
- Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
- LongViTU

### 2.2 从文献保留了什么
- 数据构造往往先有 chunk、tree 或 graph
- 这些结构决定后续问什么、答什么、引用哪些证据
- 对于 QA-first 路线，问题本身也需要被纳入统一 scaffold，而不是绕开结构层直接做补证(Question-conditioned)

## 3. 子模式定义

### 3.1 NARRATION_CHUNK
把 narration、字幕、ASR 片段与对应时间窗组织成 `narration chunk`。

### 3.2 EVENT_TREE
围绕阶段、前后依赖和事件展开关系构造 `event tree`。

### 3.3 ACTION_GRAPH
围绕动作、对象、关系和多跳连接构造 `action graph`。

### 3.4 QUERY_SCAFFOLD
围绕某条 `qa_draft` 组织候选证据，构造 query-conditioned scaffold。

它的目标不是回答问题，而是把：

- 问题草案
- 候选时间窗
- 描述单元
- 文本辅证

绑定成后续 A8 可消费的统一对象。

## 4. 输入规范

### 4.1 必需输入
以下两类输入至少满足一类：

- 结构构件类：`description_units[]`、`temporal_structures[]`
- 绑定构件类：`qa_drafts[]`、`evidence_units[]`

### 4.2 可选输入
- `text_signals[]`
- `evidence_units[]`
- `qa_drafts[]`
- `config.unit_type`
- `config.max_children_per_unit`
- `config.allow_cross_segment_link`
- `config.scaffold_role`

## 5. 输出规范
写入 `scaffold_units[]`，推荐字段：

- `scaffold_id`
- `unit_type`
- `scaffold_role`
- `source_description_ids`
- `source_temporal_unit_ids`
- `source_text_signal_ids`
- `source_evidence_unit_ids`
- `source_qa_draft_ids`
- `scaffold_summary`
- `composition_rule`
- `child_refs`
- `evidence_distribution`

### 5.1 `narration chunk` 示例

```json
{
  "scaffold_id": "narr_chunk_03",
  "unit_type": "narration_chunk",
  "scaffold_role": "generation",
  "scaffold_summary": "讲解者介绍准备材料并开始第一步操作",
  "timestamp_range": {
    "start_sec": 42.0,
    "end_sec": 57.5
  },
  "source_text_signal_ids": [
    "asr_07",
    "sub_03"
  ],
  "source_temporal_unit_ids": [
    "temp_11"
  ],
  "composition_rule": "adjacent_narration_with_temporal_alignment",
  "child_refs": [
    "asr_07",
    "sub_03",
    "temp_11"
  ]
}
```

### 5.2 `query scaffold` 示例

```json
{
  "scaffold_id": "query_pack_07",
  "unit_type": "query_scaffold",
  "scaffold_role": "binding",
  "source_qa_draft_ids": [
    "qa_draft_07"
  ],
  "source_evidence_unit_ids": [
    "ev_103",
    "ev_119",
    "ev_121"
  ],
  "scaffold_summary": "围绕杯子何时打翻这一问题组织的候选证据包",
  "composition_rule": "question_conditioned_temporal_binding",
  "child_refs": [
    "ev_103",
    "ev_119",
    "ev_121"
  ]
}
```

## 6. 与上下游的绑定关系
- 上游：A4、A5、A6
- 兼容上游：`qa_drafts[]`、`evidence_units[]`
- 下游：A8、A9、A10

## 7. 典型失败模式
- 骨架单元过大，导致问题粒度太粗
- 骨架单元过碎，导致 evidence 复用性变差
- tree / graph 关系只存文本，不存显式 child refs
- query scaffold 只挂问题，不挂候选证据
- narration scaffold 没有显式时间锚点，导致后续 grounding 强度不足

## 8. 推荐使用场景
- 默认数据构造主链路
- narration / caption-first 数据构造
- QA-first / post-hoc grounding 的证据绑定入口
- 需要规模化生成可复用 Q/A/E 样本时
