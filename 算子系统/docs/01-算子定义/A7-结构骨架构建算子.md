# 结构骨架构建算子（A7）技术文档

## 1. 在系统中的定位
A7 是当前系统默认主链路的中枢。它负责把 A4、A5、A6 的产物组织成可复用的结构骨架。

结构骨架可以是：

- narration chunk
- event tree
- action graph
- chunk graph
- 其他可共同合成的结构单元

A7 的目标不是直接生成问答，而是回答：

- 哪些 caption、时间窗和文本信号应该属于同一个单元
- 这些单元之间是什么关系
- 后续 A8 应该在哪个粒度上共同生成 `Q/A/E`

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中强`

### 2.1 文献锚点
- Grounded Question-Answering in Long Egocentric Videos
- Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
- LongViTU

### 2.2 从文献保留了什么
- 数据构造往往先有 chunk、tree 或 graph
- 这些结构决定后续问什么、答什么、引用哪些证据

## 3. 子模式定义

### 3.1 CHUNK_PACK
把相邻 caption 和时间窗组织成 chunk。

### 3.2 TREE_BUILD
构造层级树。

### 3.3 GRAPH_BUILD
构造事件图、动作图或多跳关系图。

## 4. 输入规范

### 4.1 必需输入
- `coarse_captions[]`
- `temporal_structures[]`

### 4.2 可选输入
- `text_signals[]`
- `config.unit_type`
- `config.max_children_per_unit`
- `config.allow_cross_segment_link`

## 5. 输出规范
写入 `scaffold_units[]`，推荐字段：

- `scaffold_id`
- `unit_type`
- `source_caption_ids`
- `source_temporal_unit_ids`
- `source_text_signal_ids`
- `scaffold_summary`
- `composition_rule`
- `child_refs`
- `evidence_distribution`

## 6. 与上下游的绑定关系
- 上游：A4、A5、A6
- 下游：A8、A9、A10

## 7. 典型失败模式
- 骨架单元过大，导致问题粒度太粗
- 骨架单元过碎，导致 evidence 复用性变差
- tree / graph 关系只存文本，不存显式 child refs

## 8. 推荐使用场景
- 默认数据构造主链路
- 需要规模化生成可复用 Q/A/E 样本时
