# 去重算子（A4）技术文档

## 1. 在系统中的定位
去重算子负责对采样结果进行二次压缩，删除在视觉或特征空间中高度重复的样本。它的价值不只是减少存储和推理成本，更重要的是避免后续标注算子被冗余帧主导。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`强`

### 2.1 文献锚点
- STORM
- SlowFast-LLaVA-1.5
- V-CORE
- Token Reduction via AOT

### 2.2 从文献保留了什么
- 长视频处理中必须压缩冗余视觉输入
- 压缩应尽量保留时间动态和局部关键信息
- 压缩不是纯粹删 token，而是有选择地保留代表性样本

## 3. 子模式定义

### 3.1 HASH
基于感知哈希或近似图像哈希去重。

### 3.2 FEATURE_SIM
基于视觉特征相似度去重。

### 3.3 CLUSTER_CENTER
先聚类，再保留簇中心作为代表样本。

## 4. 输入规范
消费 `samples[]`，输出仍写回 `samples[]` 并新增分组信息。

### 4.1 必需输入
- `samples[].sample_id`
- `samples[].segment_id`
- `samples[].resource_refs.frame_uri`

### 4.2 可选输入
- `config.hash_threshold`
- `config.similarity_threshold`
- `config.cluster_k`

## 5. 输出规范

### 5.1 样本级状态
- `status`
- `is_representative`
- `duplicate_group_id`

### 5.2 分组级对象
新增 `duplicate_groups[]`，推荐字段：

- `duplicate_group_id`
- `member_sample_ids`
- `representative_sample_id`
- `compression_ratio`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `hash_threshold` | 哈希距离阈值 |
| `similarity_threshold` | 特征相似阈值 |
| `cluster_k` | 聚类中心数量 |
| `max_keep_per_segment` | 每段最多保留代表帧数 |

## 7. 与上下游的绑定关系
- 上游来自 A3
- 下游 A5/A6/A7/A8 默认优先读取 `is_representative=true` 的样本

## 8. 典型失败模式
- 过度压缩，删掉细微但关键的动作变化
- 聚类中心只保留静态背景
- 跨场景误聚类，破坏时序一致性

## 9. 推荐使用场景
- 长视频
- 对话类、监控类、高冗余视频
- 后续算子成本较高时
