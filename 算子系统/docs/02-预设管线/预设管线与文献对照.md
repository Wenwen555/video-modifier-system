# 预设管线与文献对照

## 1. 文档目标
本文件用于把若干经典文献工作的核心处理链路改写为当前算子系统中的预设表达。

这里关注的是：

- 哪些论文工作最值得作为对外展示的 preset
- 它们在当前算子系统里分别对应哪些算子组合
- 哪些环节属于文献原生能力，哪些环节属于系统化补全


## 2. 说明
下面的 preset 都是“论文核心思路的算子化重写”，而不是逐行复现原论文或原仓库代码。

### 2.1 辅助节点
为便于对齐 Panda-70M 这类 split-and-stitch 结构，本页允许使用一个桥接节点：

- `MergeOp`

它表示：

- 位于 `A1` 之后
- 负责把候选短片段合并为 event 级片段
- 用于表达文献中的语义拼接或相邻片段合并逻辑
- 不计入当前固定 10 算子

## 3. 预设总表

| 文献工作 | 预设链路 | 核心关注 | 最适合展示的系统能力 |
|---|---|---|---|
| `Panda-70M` | `A1 -> MergeOp -> A2 -> A5` | 分层切分、相邻合并、事件级清洗 | A1 的 proposal 思想和 metadata-first 设计 |
| `Video-ChatGPT / Video-LLaVA` | `A1 -> A2 -> A3 -> A9 -> A10` | 基础多帧采样后直接生成 | A3 作为经典视频输入基线 |
| `TimeChat-Captioner` | `A1 -> A3 -> A6 -> A8 -> A9 -> A10` | 显式时间结构、带时间边界的生成 | A6+A8+A9 的时间感知标注链 |
| `Video-in-the-Loop` | `A1 -> A3 -> A4 -> A8 -> A9 -> A10` | 预算重分配、关键 span 聚焦 | A4 的自适应采样和 A8 的时序定位 |
| `SpaceVLLM / VideoLoom` | `A1 -> A3/A4 -> A7 -> A8 -> A9 -> A10` | 空间 grounding 与时间 grounding 联动 | A7+A8 的空间时序联合证据 |
| `VDC-Agent / LongViTU` | `A1 -> A2 -> A3 -> A6 + A7 + A8 -> A9 -> A10` | 多证据标注生成与质量裁决 | A9+A10 的闭环式标注构建 |

## 4. 典型预设

### 4.1 Panda-70M
链路：
`A1 -> MergeOp -> A2 -> A5`

说明：

- `A1` 负责候选切分，强调高召回 proposal
- `MergeOp` 负责相邻片段的语义拼接
- `A2` 负责 event 级质量控制和清洗
- `A5` 负责进一步压缩重复或近重复片段

在Panda-70M中，其splitting的设计原则：

- 划分不是一次性最终判定，而是 proposal + merge + verify
- 切分决策应优先输出 metadata，而不是直接产出物化视频片段

### 4.2 Video-ChatGPT / Video-LLaVA
链路：

`A1 -> A2 -> A3 -> A9 -> A10`

说明：

- `A1` 保证输入时间范围可消费
- `A3` 提供基础多帧采样
- `A9` 直接在基础采样之上生成 caption 或 QA
- `A10` 负责后验筛选
：

多帧均匀或轻量采样仍然是视频理解的标准基线入口，该 preset 是最容易跑通的基础方案

### 4.3 TimeChat-Captioner
链路：

`A1 -> A3 -> A6 -> A8 -> A9 -> A10`

说明：

- `A3` 提供片段级基础视觉样本
- `A6` 汇聚字幕、OCR、ASR 等文本证据
- `A8` 显式建模时间边界和变化片段
- `A9` 产出带时间结构的 caption 或 script

这篇文献给的启发：

- 长视频标注不能只依赖静态 caption
- 时间结构和文本证据应显式进入生成链路

### 4.4 Video-in-the-Loop
链路：

`A1 -> A3 -> A4 -> A8 -> A9 -> A10`

说明：

- `A3` 先做低成本全局浏览
- `A4` 再把预算集中到更关键的片段或时刻
- `A8` 输出关键时间段和事件结构
- `A9` 在精化后的局部上下文上生成结果

这篇文献给的启发：

- 采样预算不是平均分配的
- 长视频处理的关键是 span-aware 的资源重分配

### 4.5 SpaceVLLM / VideoLoom
链路：

`A1 -> A3/A4 -> A7 -> A8 -> A9 -> A10`

说明：

- `A3/A4` 提供基础或自适应视觉入口
- `A7` 生成对象级、区域级空间证据
- `A8` 把空间对象挂接到时间边界和变化事件
- `A9` 生成 grounded caption 或时空问答结果

这篇文献给的启发：

- 空间 grounding 和时间 grounding 应联合建模
- 这类 preset 更适合展示系统的证据链完整性

### 4.6 VDC-Agent / LongViTU
链路：

`A1 -> A2 -> A3 -> A6 + A7 + A8 -> A9 -> A10`

说明：

- `A6`、`A7`、`A8` 共同形成多证据分支
- `A9` 在多证据基础上生成候选标注
- `A10` 负责多维质量裁决和最终保留决策

这篇文献给的启发：

- 标注生成不应脱离证据
- 质量裁决需要成为系统闭环的一部分

## 5. 展示顺序

1. `Panda-70M`
2. `Video-ChatGPT / Video-LLaVA`
3. `TimeChat-Captioner`
4. `Video-in-the-Loop`
5. `SpaceVLLM / VideoLoom`
6. `VDC-Agent / LongViTU`

这个顺序的逻辑是：先展示切分骨架，再展示基础采样基线、时间结构增强、预算的重分配，以及最后展示时空 grounding 和质量裁决闭环
