# 预设管线与文献对照

## 1. 文档目标
本文件用于把若干经典文献工作的核心处理链路改写为当前算子系统中的预设表达。

这里关注的是：

- 哪些论文工作最值得作为对外展示的 preset
- 它们在当前算子系统里分别对应哪些算子组合
- 哪些环节属于文献原生能力，哪些环节属于系统化补全

## 2. 说明
下面的 preset 都是“论文核心思路的算子化重写”，而不是逐行复现原论文或原仓库代码。

### 2.1 辅助节点
为便于对齐 Panda-70M 这类 split-and-stitch 结构，本页允许使用一个桥接节点：

- `MergeOp`

它表示：

- 位于 `A1` 之后
- 负责把候选短片段合并为 event 级片段
- 用于表达文献中的语义拼接或相邻片段合并逻辑
- 不计入当前固定 9 算子

### 2.2 关于采样
原系统中的 `A3 基础采样` 与 `A4 自适应采样` 已合并为新的 `A3 采样算子`。因此本页凡是涉及“先粗采样，再重分配预算”的论文，都统一写作 `A3` 的不同模式或两阶段内部策略。

## 3. 预设总表

| 文献工作 | 预设链路 | 核心关注 | 最适合展示的系统能力 |
|---|---|---|---|
| `Panda-70M` | `A1 -> MergeOp -> A2 -> A3` | 分层切分、相邻合并、上下文单元构造 | A1+A2 的 split-and-stitch 表达 |
| `Video-ChatGPT / Video-LLaVA` | `A1 -> A2 -> A3 -> A7 -> A8 -> A9` | 基础多帧采样后直接产出问答 | A3 作为经典视频输入基线 |
| `TimeChat-Captioner` | `A1 -> A2 -> A3 -> A4 -> A6 -> A7 -> A8 -> A9` | 显式时间结构与文本辅证融合 | A4+A6+A7 的时间感知 QA 规划链 |
| `Video-in-the-Loop` | `A1 -> A2 -> A3 -> A4 -> A7 -> A8 -> A9` | 预算重分配、关键 span 聚焦 | A3 的自适应采样和 A4 的时序定位 |
| `SpaceVLLM / VideoLoom` | `A1 -> A2 -> A3 -> A5 -> A7 -> A8 -> A9` | 空间 grounding 驱动问答生成 | A5+A7 的空间证据规划链 |
| `VDC-Agent / LongViTU` | `A1 -> A2 -> A3 -> A4 + A5 + A6 -> A7 -> A8 -> A9` | 多证据规划、生成与 grounded 回修闭环 | A7+A8+A9 的闭环式数据构建 |

## 4. 典型预设

### 4.1 Panda-70M
链路：
`A1 -> MergeOp -> A2 -> A3`

说明：

- `A1` 负责候选切分，强调高召回 proposal
- `MergeOp` 负责相邻片段的语义拼接
- `A2` 负责把拼接后的 event 片段整理为稳定上下文单元
- `A3` 负责提供后续可消费的代表帧或片段样本

在 Panda-70M 中，其 splitting 的设计原则：

- 划分不是一次性最终判定，而是 proposal + merge + verify
- 切分决策应优先输出 metadata，而不是直接产出物化视频片段

### 4.2 Video-ChatGPT / Video-LLaVA
链路：

`A1 -> A2 -> A3 -> A7 -> A8 -> A9`

说明：

- `A1` 保证输入时间范围可消费
- `A2` 保证多帧输入来自稳定上下文单元
- `A3` 提供基础多帧采样
- `A7` 做轻量问题规划
- `A8` 直接在基础采样之上生成 QA
- `A9` 负责 grounded 后验筛选

多帧均匀或轻量采样仍然是视频理解的标准基线入口，该 preset 是最容易跑通的基础方案。

### 4.3 TimeChat-Captioner
链路：

`A1 -> A2 -> A3 -> A4 -> A6 -> A7 -> A8 -> A9`

说明：

- `A3` 提供片段级视觉样本
- `A4` 显式建模时间边界和变化片段
- `A6` 汇聚字幕、OCR、ASR 等文本辅证
- `A7` 基于时序与文本证据规划问题
- `A8` 产出带时间结构的 QA

这篇文献给的启发：

- 长视频标注不能只依赖静态 caption
- 时间结构和文本证据应显式进入生成链路

### 4.4 Video-in-the-Loop
链路：

`A1 -> A2 -> A3 -> A4 -> A7 -> A8 -> A9`

说明：

- `A3` 内部可先做低成本全局浏览，再把预算集中到更关键的片段或时刻
- `A4` 输出关键时间段和事件结构
- `A7` 把关键 span 转成明确的问题计划
- `A8` 在精化后的局部上下文上生成结果
- `A9` 负责最终 grounded 核验与回修

这篇文献给的启发：

- 采样预算不是平均分配的
- 长视频处理的关键是 span-aware 的资源重分配
- 这类能力应被视为 `A3` 的高阶模式，而不是额外拆出的新阶段

### 4.5 SpaceVLLM / VideoLoom
链路：

`A1 -> A2 -> A3 -> A5 -> A7 -> A8 -> A9`

说明：

- `A3` 提供基础或自适应视觉入口
- `A5` 生成对象级、区域级空间证据
- `A7` 把空间对象挂接到问题意图和答案形式
- `A8` 生成 grounded QA 或时空问答结果

这篇文献给的启发：

- 空间 grounding 和时间 grounding 应联合建模
- 这类 preset 更适合展示系统的证据链完整性

### 4.6 VDC-Agent / LongViTU
链路：

`A1 -> A2 -> A3 -> A4 + A5 + A6 -> A7 -> A8 -> A9`

说明：

- `A4`、`A5`、`A6` 共同形成多证据分支
- `A7` 先在多证据基础上规划问题
- `A8` 再生成候选 QA
- `A9` 负责多维 grounded 核验、回修和最终保留决策

这篇文献给的启发：

- 标注生成不应脱离证据
- 质量裁决需要成为系统闭环的一部分

## 5. 展示顺序

1. `Panda-70M`
2. `Video-ChatGPT / Video-LLaVA`
3. `TimeChat-Captioner`
4. `Video-in-the-Loop`
5. `SpaceVLLM / VideoLoom`
6. `VDC-Agent / LongViTU`

这个顺序的逻辑是：先展示切分骨架，再展示基础采样基线、时间结构增强、预算的重分配，以及最后展示时空 grounding 和质量裁决闭环。
