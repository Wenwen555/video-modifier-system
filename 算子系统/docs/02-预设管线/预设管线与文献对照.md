# 预设管线与文献对照

## 1. 文档目标
本文件用于把若干经典文献工作的核心处理链路改写为当前算子系统中的预设表达。

这里关注的是：

- 哪些论文工作最值得作为对外展示的 preset
- 它们在当前算子系统里分别对应哪些算子组合
- 哪些环节属于文献原生能力，哪些环节属于系统化补全

## 2. 当前默认 preset

当前系统默认主链路是：

`A1 -> A2 -> A3 -> A4 -> A5/A6 -> A7 -> A8 -> A9 -> A10`

高层含义是：

`划分 -> 编排 -> 采样 -> 粗描述 -> 时间/文本结构 -> 结构骨架 -> Q/A/E 共同合成 -> 一致性回修 -> 质量筛查`

## 3. 预设总表

| 文献工作 | 路径归类 | 预设链路 | 核心关注 | 最适合展示的系统能力 |
|---|---|---|---|---|
| `Panda-70M` | 基础骨架 | `A1 -> MergeOp -> A2 -> A3` | 分层切分、相邻合并、上下文单元构造 | A1+A2 的 split-and-stitch 表达 |
| `TimeChat-Captioner` | 结构提取 | `A1 -> A2 -> A3 -> A4 -> A5 -> A6 -> A7` | 时间结构化描述与文本辅证组织 | A4+A5+A6+A7 的骨架前处理 |
| `Grounded Question-Answering in Long Egocentric Videos` | 路径A | `A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A10` | narration chunk 到 `(Q, A, T)` 的共同构造 | scaffold-first + joint synthesis |
| `Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos` | 路径A | `A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A9 -> A10` | action scene graph、多跳 triplets、过滤与 refinement | graph scaffold + multihop QAE |
| `LongViTU` | 路径A | `A1 -> A2 -> A3 -> A4 -> A5 -> A7 -> A8 -> A9 -> A10` | hierarchical tree、timestamped QA、自回修 | tree scaffold + synthesis + revision |
| `Pinpointing Trigger Moment for Grounded Video QA` | 路径B | `外部Question -> 推理/QA -> grounding -> tracking` | 先 reasoning，再 grounding | 作为系统旁路理解 |
| `Video-in-the-Loop` | 路径C | `外部Question -> localization -> 局部重看 -> answer` | ground-first 推理 | 作为系统旁路理解 |

## 4. 典型预设说明

### 4.1 Grounded Question-Answering in Long Egocentric Videos
推荐映射：

`A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A10`

理由：

- 这篇工作先组织 narration chunks
- 再从 chunk 共同生成 `(Q, A, T)`
- 更接近 `temporal scaffold -> joint synthesis`

### 4.2 Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
推荐映射：

`A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A9 -> A10`

理由：

- 这篇工作先从 narrations 构造 action scene graph
- 再在 graph 上生成 `question-answer-evidence triplets`
- 后面还有 filtering 与人工 refinement

### 4.3 LongViTU
推荐映射：

`A1 -> A2 -> A3 -> A4 -> A5 -> A7 -> A8 -> A9 -> A10`

理由：

- 它先构造 hierarchical video tree
- 再在 subtree 上生成带时间戳的 QA
- 最后有 self-revision 机制

## 5. 展示顺序

1. `Panda-70M`
2. `TimeChat-Captioner`
3. `Grounded Question-Answering in Long Egocentric Videos`
4. `Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos`
5. `LongViTU`
6. `Pinpointing Trigger Moment for Grounded Video QA`
7. `Video-in-the-Loop`

这个顺序的逻辑是：先展示结构骨架的前处理，再展示共同合成式数据构造主链，最后把其他 grounded QA 路线作为旁路对照。
