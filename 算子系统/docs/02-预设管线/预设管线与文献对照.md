# 预设管线与文献对照

## 1. 文档目标
本文件用于把若干经典文献工作的核心处理链路改写为当前算子系统中的预设表达。

这里关注的是：

- 哪些论文工作最值得作为对外展示的 preset
- 它们在当前算子系统里分别对应哪些算子组合
- 哪些环节属于文献原生能力，哪些环节属于系统化补全

## 2. 说明
下面的 preset 都是“论文核心思路的算子化重写”，而不是逐行复现原论文或原仓库代码。

### 2.1 辅助节点
为便于对齐 Panda-70M 这类 split-and-stitch 结构，本页允许使用一个桥接节点：

- `MergeOp`

它表示：

- 位于 `A1` 之后
- 负责把候选短片段合并为 event 级片段
- 用于表达文献中的语义拼接或相邻片段合并逻辑
- 不计入当前固定 9 算子

### 2.2 关于局部重看
部分 route-two 文献会在 A4 之后触发局部重采样或 span-aware reallocation。为了不把它误写成新算子，本页统一把它视为：

- `A3` 的局部重看策略

### 2.3 当前默认主链路
当前系统默认主链路不是所有论文的唯一模板，但它是网站当前落地版的默认 preset：

`A1 -> A2 -> A3 -> A7 -> A8 -> A4 -> [A5 + A6] -> A9`

其中：

- `A7` 负责粗 caption、QA 草案与 claim slots
- `A8` 负责把草案或外部问题改写为 query bundle
- `A9` 内部负责证据摘要、答案回修、严格审查和最终筛查

## 3. 预设总表

| 文献工作 | 路径归类 | 预设链路 | 核心关注 | 最适合展示的系统能力 |
|---|---|---|---|---|
| `Panda-70M` | 基础骨架 | `A1 -> MergeOp -> A2 -> A3` | 分层切分、相邻合并、上下文单元构造 | A1+A2 的 split-and-stitch 表达 |
| `Video-ChatGPT / Video-LLaVA` | 轻量基线 | `A1 -> A2 -> A3 -> A7 -> A9(light)` | 基础多帧采样后先产出草案或轻量答案 | A3+A7 的低成本生成基线 |
| `TimeChat-Captioner` | 路径一近邻 | `A1 -> A2 -> A3 -> A7 -> A8 -> A4 -> A6 -> A9` | 时间结构化草案 + 文本辅证回看 | A4+A6 的时间感知 grounded 闭环 |
| `Pinpointing Trigger Moment for Grounded Video QA` | 路径一 | `A1 -> A2 -> A3 -> A7 -> A8 -> A4 -> A5 -> A9` | 先 reasoning / QA，再做时空 grounding | 路线一默认主链路的代表 |
| `Video-in-the-Loop` | 路径二 | `A1 -> A2 -> A3 -> 外部Question -> A8 -> A4 -> A3(局部重看) -> A9` | 预算重分配、关键 span 聚焦 | A3 的局部重看和 A4 的时序定位 |
| `Can I Trust Your Answer?` | 路径二 | `A1 -> A2 -> A3 -> 外部Question -> A8 -> A4 -> A9` | 先时序定位，再约束答案可信度 | A4-only 的 temporal grounding 优先版 |
| `SpaceVLLM / VideoLoom` | 路径二扩展 | `A1 -> A2 -> A3 -> 外部Question或对象提示 -> A8 -> A4 -> A5 -> A9` | 空间 grounding 驱动答案生成 | A5 的局部证据闭环 |
| `VDC-Agent / LongViTU` | 路径一增强 | `A1 -> A2 -> A3 -> A7 -> A8 -> A4 + A5 + A6 -> A9` | 多证据草案、回看、回修闭环 | A7+A8+A9 的数据构建闭环 |
| `MUPA` | 路径三 | `A1 -> A2 -> A3 -> [路径A] + [路径B] -> Aggregation` | 多路径协同与反思裁决 | 路径一与路径二并行保留 |

## 4. 典型预设

### 4.1 Panda-70M
链路：

`A1 -> MergeOp -> A2 -> A3`

说明：

- `A1` 负责候选切分，强调高召回 proposal
- `MergeOp` 负责相邻片段的语义拼接
- `A2` 负责把拼接后的 event 片段整理为稳定上下文单元
- `A3` 负责提供后续可消费的代表帧或片段样本

### 4.2 Video-ChatGPT / Video-LLaVA
链路：

`A1 -> A2 -> A3 -> A7 -> A9(light)`

说明：

- 这类工作没有完整的 grounded 回看闭环
- 在当前系统中，更适合作为“轻量草案生成基线”展示
- `A9(light)` 表示只做轻量过滤，不强行补全 route-one 的全部取证节点

### 4.3 TimeChat-Captioner
链路：

`A1 -> A2 -> A3 -> A7 -> A8 -> A4 -> A6 -> A9`

说明：

- `A7` 先形成时间感知的粗草案
- `A8` 把草案改写成更适合 temporal/textual grounding 的 query
- `A4` 与 `A6` 共同支撑时间结构与文本线索

### 4.4 Pinpointing Trigger Moment for Grounded Video QA
链路：

`A1 -> A2 -> A3 -> A7 -> A8 -> A4 -> A5 -> A9`

说明：

- 它最接近当前网站默认路线一
- 先有 reasoning / QA 草案，再回看做 spatio-temporal grounding
- A9 再把证据组织成最终成品结果

### 4.5 Video-in-the-Loop
链路：

`A1 -> A2 -> A3 -> 外部Question -> A8 -> A4 -> A3(局部重看) -> A9`

说明：

- `A8` 把 question 改写为可执行 query
- `A4` 负责找到关键 span
- `A3` 再做局部重看或预算重分配
- `A9` 基于更高质量局部上下文完成最终输出

### 4.6 Can I Trust Your Answer?
链路：

`A1 -> A2 -> A3 -> 外部Question -> A8 -> A4 -> A9`

说明：

- 它是路径二里更纯粹的 temporal grounding-first 版本
- 空间分支不是必需默认项
- 更强调先定位，再约束答案的可信性

### 4.7 VDC-Agent / LongViTU
链路：

`A1 -> A2 -> A3 -> A7 -> A8 -> A4 + A5 + A6 -> A9`

说明：

- `A7` 和 `A8` 共同构成草案与 query 中间层
- `A4`、`A5`、`A6` 提供多路证据回看
- `A9` 负责 grounded 回修、审查和最终保留决策

## 5. 展示顺序

1. `Panda-70M`
2. `Video-ChatGPT / Video-LLaVA`
3. `TimeChat-Captioner`
4. `Pinpointing Trigger Moment for Grounded Video QA`
5. `Video-in-the-Loop`
6. `Can I Trust Your Answer?`
7. `SpaceVLLM / VideoLoom`
8. `VDC-Agent / LongViTU`
9. `MUPA`

这个顺序的逻辑是：先展示切分骨架，再展示轻量生成基线、路径一默认链路、路径二推理链路，最后展示多证据闭环和多路径聚合。
