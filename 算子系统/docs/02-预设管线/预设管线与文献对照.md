# 预设管线与文献对照

## 1. 文档目标
本文件用于把若干经典文献工作的核心处理链路改写为当前算子系统中的预设表达。

这里关注的是：

- 哪些论文工作最值得作为对外展示的 preset
- 它们在当前算子系统里分别对应哪些算子组合
- 哪些环节属于文献原生能力，哪些环节属于系统化补全

当前所有 preset 的共同目标是：

- 生成适合 `post-training / instruction tuning` 的高质量 LVLM QA 数据
- 在统一协议下兼容不同数据构造路线
- 让最终样本都能收敛到统一的 `qae_triplets -> revisions -> quality_records` 出口

## 2. 当前默认 preset

当前系统默认主链路是：

`A1 -> A2 -> A3 -> A4 -> A5/A6 -> A7 -> A8 -> A9 -> A10`

高层含义是：

`划分 -> 轻量编排 -> 采样 -> 多粒度描述 -> 时间/文本结构 -> 证据规约 / 结构骨架 -> Q/A/E 物化 -> 一致性回修 -> 质量筛查`

它当前对应的是：

- 路由家族：`structure_first`
- 训练目标：`grounded_qa_instruction_tuning`
- 数据定位：高质量、可回溯、可筛查的 LVLM post-training 数据

## 5. 典型预设说明

### 5.1 Grounded Question-Answering in Long Egocentric Videos
推荐映射：

`A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A10`

理由：

- 这篇工作先组织 narration chunks
- 再从 chunk 共同生成 `(Q, A, T)`
- 更接近 `temporal scaffold -> joint synthesis`

### 5.2 Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos
推荐映射：

`A1 -> A2 -> A3 -> A5 -> A6 -> A7 -> A8 -> A9 -> A10`

理由：

- 这篇工作先从 narrations 构造 action scene graph
- 再在 graph 上生成 `question-answer-evidence triplets`
- 后面还有 filtering 与人工 refinement

### 5.3 LongViTU
推荐映射：

`A1 -> A2 -> A3 -> A4 -> A5 -> A7 -> A8 -> A9 -> A10`

理由：

- 它先构造 hierarchical video tree
- 再在 subtree 上生成带时间戳的 QA
- 最后有 self-revision 机制

### 5.4 QA-First / Post-hoc Grounding Preset
推荐映射：

`外部QA草案 -> A1 -> A2 -> A3 -> A4/A5/A6 -> A7 -> A8 -> A9 -> A10`

理由：

- 先导入人工或外部 QA
- 再在视频上抽取候选 temporal / textual / structural evidence
- 由 `A7` 组织成 query scaffold
- 由 `A8` 定稿为带 evidence 的最终 triplets

## 6. 展示顺序

1. `Panda-70M`
2. `TimeChat-Captioner`
3. `Grounded Question-Answering in Long Egocentric Videos`
4. `Grounded Multi-Hop VideoQA in Long-Form Egocentric Videos`
5. `LongViTU`
6. `Pinpointing Trigger Moment for Grounded Video QA`
7. `Video-in-the-Loop`

这个顺序的逻辑是：

- 先展示结构骨架的前处理
- 再展示三条数据构造主线如何在系统中收敛
- 最后把 ground-first 推理路线作为旁路对照
