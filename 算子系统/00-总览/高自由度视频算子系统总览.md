# 高自由度视频算子系统总览

## 1. 文档定位
本文件是 `算子系统/` 目录下的总设计文档，用于回答三个核心问题：

1. 这套系统为什么不是简单地堆更多工具，而是围绕一条标准数据流来组织。
2. 为什么当前版本固定为 10 个视频算子，而不是继续无限扩张。
3. 哪些算子是文献强驱动的，哪些算子是为了把整条增强链路真正跑通而保留的系统支撑层。

本设计严格限定在 `算子系统/` 文件夹内部，不引入其他外部框架名称作为系统主轴。

## 2. 设计目标

### 2.1 高自由度
系统要同时支持两种使用方式：

- 原子化自由拼装：研究者可以按任务需要自由组合算子。
- 预设模板复用：可以快速复现若干论文风格或场景风格的处理链路。

### 2.2 覆盖完整视频增强链路
首版 10 个算子必须覆盖从视频切分到最终标注与筛选的完整流程：

`segment -> clean -> sample -> dedup/compress -> text evidence -> spatial focus -> temporal summarize -> annotation -> quality gate`

### 2.3 强调数据流而不是单点技巧
算子系统的价值不在于某一个算子本身多么先进，而在于：

- 上游产物能否被下游稳定消费
- 中间态是否统一
- 数据血缘是否可追溯
- 同一段视频是否能在不同预算下被重用

### 2.4 与文献对齐
系统不追求把每一篇论文原封不动封装成一个黑盒，而是从 LVLM / Video-LLM 文献中提炼出稳定、可复用的原子能力，并明确哪些能力具有强文献支撑。

## 3. 10 个视频算子

| 编号 | 算子名称 | 核心职责 | 文献依赖等级 | 最新支撑强度 |
|---|---|---|---|---|
| A1 | 划分算子 | 把原视频切成时序一致的片段 | P1 论文拆解 | 中 |
| A2 | 片段清洗算子 | 过滤低质量、低信息密度片段 | P0 系统支撑 | 弱 |
| A3 | 采样算子 | 进行低成本基础采样 | P2 论文原生 | 强 |
| A4 | 自适应采样算子 | 内容感知的高级采样 | P2 论文原生 | 很强 |
| A5 | 去重算子 | 压缩冗余帧和冗余视觉表征 | P1 论文拆解 | 强 |
| A6 | 文本抽取算子 | 收集 OCR、字幕、ASR 等文本证据 | P1 论文拆解 | 中偏弱 |
| A7 | 空间聚焦算子 | 生成区域级视觉证据 | P2 论文原生 | 很强 |
| A8 | 时序定位与变化摘要算子 | 建模关键时刻和变化边界 | P2 论文原生 | 很强 |
| A9 | 标注生成算子 | 生成 caption、QA、summary 等标注 | P2 论文原生 | 很强 |
| A10 | 候选选择与质量裁决算子 | 选择优质候选并做最终过滤 | P2 论文原生 | 中强 |

文献依赖等级说明：

- `P2 论文原生`：该能力能直接映射到已有文献中的明确处理环节。
- `P1 论文拆解`：不是论文中的独立模块，但可稳定拆为原子算子。
- `P0 系统支撑`：主要服务于整条链路闭环，不主张作为某篇论文的直接复刻。

## 4. 系统分层

### 4.1 基础系统层
负责建立稳定的数据流入口和预算可控的基础处理能力。

- A1 划分算子
- A2 片段清洗算子
- A3 采样算子
- A5 去重算子
- A6 文本抽取算子

### 4.2 文献强驱动层
负责提供当前 Video-LLM 研究真正强调的高价值能力。

- A4 自适应采样算子
- A7 空间聚焦算子
- A8 时序定位与变化摘要算子
- A9 标注生成算子
- A10 候选选择与质量裁决算子

## 5. 主数据流
系统主干数据流固定为：

`Raw Video -> A1 -> A2 -> A3/A4 -> A5 -> A6/A7/A8 -> A9 -> A10`

并行关系说明：

- `A3` 和 `A4` 可以二选一，也可以同时执行
- `A5` 负责对多路采样结果进行统一压缩或保留
- `A6`、`A7`、`A8` 在采样完成后并行提取不同维度证据
- `A9` 负责把多路证据整理成可训练标注
- `A10` 是唯一的最终出口

## 6. 系统设计约束

### 6.1 统一中间态
所有算子都必须通过同一份 `VideoPacket` 交互，不允许每个算子定义自己的私有协议。

### 6.2 追加式写入
算子只允许追加新对象或追加新状态，不允许覆写上游原始结果。

### 6.3 证据先于文本
任何最终标注都必须指向明确证据，不能让标注生成算子直接对原视频自由发挥。

### 6.4 模式与预算解耦
每个算子至少应该区分：

- 轻量模式：用于跑通链路和低成本批处理
- 平衡模式：用于通用网页混合视频
- 重模式：用于高质量样本生产

## 7. 推荐预设

### 7.1 Lite
链路：

`A1(SHOT) -> A2 -> A3(CENTER) -> A6 -> A9(SEGMENT_CAPTION) -> A10`

适用：

- 快速打底
- 短视频批处理
- 低算力环境

### 7.2 Balanced
链路：

`A1(HYBRID) -> A2 -> A3 + A4 -> A5 -> A6 + A7 + A8 -> A9 -> A10`

适用：

- 通用网页混合视频
- 默认实验链路
- 汇报展示中的主推方案

### 7.3 Heavy
链路：

`A1(SEMANTIC/HYBRID) -> A2(strict) -> A4 -> A5 -> A6 + A7 + A8 -> A9 -> A10(strict)`

适用：

- 高价值样本生产
- 长视频或复杂时序视频
- 强调空间与时序对齐的标注构建

## 9. 当前版本的取舍
这套设计刻意没有把调度器、缓存、存储、分布式执行器纳入 10 个算子，因为这些属于系统基础设施而不是视频增强算子本身。

当前版本也没有把音频事件检测单独拆为新算子，而是把字幕、ASR、OCR 等统一归入 A6 文本抽取算子。这样可以在保证系统完整性的前提下控制复杂度，避免第一版过度膨胀。
