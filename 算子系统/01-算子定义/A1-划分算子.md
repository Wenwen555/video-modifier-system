# 划分算子（A1）技术文档

## 1. 在系统中的定位
划分算子是整条视频增强链路的入口。它负责把原始连续视频切成可以被后续算子独立消费的片段，为后续采样、文本抽取、时序建模和标注生成提供最小处理单元。

它解决的不是“理解视频”，而是“决定后面所有算子应该在哪些时间范围上工作”。

## 2. 文献依赖等级

- 文献依赖等级：`P1 论文拆解`
- 最新支撑强度：`中`

### 2.1 文献锚点
- Panda-70M：强调先获得语义或视觉一致的片段，再进行后续标注生成。
- TimeChat-Captioner：强调长视频需要显式的时间段结构来支撑带时间戳的脚本化描述。
- Video-in-the-Loop：强调先定位关键时间范围，再把计算预算集中到相关 span。

### 2.2 从文献保留了什么
- 先做局部时间范围限定，再做后续理解。
- 切分结果必须保留时间边界和置信度。
- 切分模式不应只有物理镜头，还应支持语义层切分。

### 2.3 为了原子化做了什么拆解
很多最新论文不把“划分”单独命名为独立模块，而是把它折叠进 semantic retrieval、temporal grounding 或 caption scheduling 中。本系统将其前置为独立算子，是为了让不同下游链路共享同一份片段划分结果。

## 3. 子模式定义

### 3.1 FIXED
按固定时间窗口或固定帧数切分。

适合：

- 实时性要求高
- 只需要轻量基线
- 后续会再做强采样和强过滤

### 3.2 SHOT
基于镜头边界检测的物理切分。

适合：

- 影视剧
- 混剪视频
- 视觉转场明显的网页视频

### 3.3 SEMANTIC
基于视觉特征、字幕或文本信号的语义事件切分。

适合：

- 长教程
- 新闻视频
- 一镜到底但内部语义变化明显的视频

### 3.4 HYBRID
先做 shot 级粗切，再做 segment merge 或 semantic refine。

适合：

- 通用网页混合视频
- 作为系统默认推荐模式

## 4. 输入规范
划分算子消费 `VideoPacket.video` 和运行配置，输出 `VideoPacket.segments`。

### 4.1 必需输入
- `video.video_id`
- `video.video_uri`
- `video.metadata.fps`
- `video.metadata.total_frames`
- `video.metadata.duration_sec`
- `config.partition_mode`

### 4.2 可选输入
- `video.subtitle_uri`
- `video.asr_uri`
- `config.chunk_duration_sec`
- `config.detector_type`
- `config.sensitivity_threshold`
- `config.similarity_threshold`
- `config.modality_weights`

## 5. 输出规范
划分算子追加 `segments[]`，每个 `segment` 至少包含：

- `segment_id`
- `source_ids`
- `timestamp_range.start_sec`
- `timestamp_range.end_sec`
- `frame_range.start_frame`
- `frame_range.end_frame`
- `duration_sec`
- `boundary_type`
- `confidence`
- `status`

推荐附加字段：

- `partition_mode`
- `semantic_shift_score`
- `visual_change_score`
- `text_shift_score`
- `resource_refs.preview_frame_uri`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `chunk_duration_sec` | 固定切分窗口 |
| `sensitivity_threshold` | 控制镜头切分灵敏度 |
| `similarity_threshold` | 控制语义切分触发阈值 |
| `min_segment_duration_sec` | 避免过短碎片 |
| `max_segment_duration_sec` | 对超长片段进行限制 |
| `merge_adjacent_small_segments` | 合并极短邻接片段 |

## 7. 与上下游的绑定关系

### 7.1 上游
无视频算子上游。它直接消费原始视频及其元数据。

### 7.2 下游
- A2 片段清洗算子直接消费 `segments`
- A3/A4 采样算子按 `segment_id` 工作
- 所有后续标注与证据都必须回指到 `segment_id`

## 8. 典型失败模式

### 8.1 过切
把一个完整动作拆成大量短片段，导致采样和标注失去上下文。

### 8.2 欠切
一个片段内部混入多个不同事件，后续采样难以兼顾。

### 8.3 转场污染
镜头边界识别不稳，导致转场模糊帧被保留。

### 8.4 语义漂移漏检
长镜头内部发生明显语义变化，但没有触发二次切分。

## 9. 推荐使用场景
- 短视频或影视混剪：优先 `SHOT`
- 长教程或直播切片：优先 `SEMANTIC`
- 通用网页混合视频：优先 `HYBRID`
- 只做轻量基线：使用 `FIXED`
