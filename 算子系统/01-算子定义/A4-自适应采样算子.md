# 自适应采样算子（A4）技术文档

## 1. 在系统中的定位
自适应采样算子负责在固定预算下，把采样资源集中到真正重要的时间点和信息变化点。它是从“盲采样”走向“内容感知采样”的关键一步，也是当前 Video-LLM 文献中最强势的方向之一。

## 2. 文献依赖等级

- 文献依赖等级：`P2 论文原生`
- 最新支撑强度：`很强`

### 2.1 文献锚点
- Adaptive Keyframe Sampling
- STORM
- Threading Keyframe with Narratives
- AdaRD-Key
- Video-in-the-Loop
- Think with Grounding

### 2.2 从文献保留了什么
- 采样必须对信息密度变化敏感
- 固定预算下需要兼顾 relevance 和 diversity
- 关键时间范围应被分配更多视觉预算

### 2.3 为了原子化做了什么拆解
最新论文常把自适应采样与检索、re-grounding、token reallocation 写在一起。本系统将其收敛为独立算子，保证它能在不同 pipeline 中被复用。

## 3. 子模式定义

### 3.1 KEYFRAME
基于特征代表性选择关键帧。

### 3.2 FEATURE_DRIFT
基于相邻帧语义距离或特征漂移触发采样。

### 3.3 MOTION_AWARE
基于运动强度、轨迹变化或局部动态信息触发采样。

## 4. 输入规范
消费 `segments[]`，追加 `samples[]`。

### 4.1 必需输入
- `segments[].segment_id`
- `segments[].frame_range`
- `config.mode`

### 4.2 可选输入
- `config.encoder_type`
- `config.max_frames`
- `config.redundancy_threshold`
- `config.semantic_drift_threshold`
- `config.motion_threshold`

## 5. 输出规范
输出写入 `samples[]`，推荐字段：

- `sample_id`
- `segment_id`
- `sample_type`
- `timestamp_sec`
- `frame_index`
- `sample_reason`
- `information_score`
- `confidence`

## 6. 关键参数

| 参数 | 作用 |
|---|---|
| `max_frames` | 每段最多保留帧数 |
| `redundancy_threshold` | 控制相似帧过滤 |
| `semantic_drift_threshold` | 控制语义漂移触发 |
| `motion_threshold` | 控制运动触发 |
| `encoder_type` | 指定视觉编码器 |

## 7. 与上下游的绑定关系

### 7.1 上游
- A1 提供片段边界
- A2 过滤明显无效片段

### 7.2 下游
- A5 可以对自适应采样结果再压缩
- A7 和 A8 优先消费自适应采样结果
- A9 可把 `sample_reason` 作为标注辅助信息

## 8. 典型失败模式
- 只关注剧烈变化，忽略稳定但重要的语义帧
- 特征编码器不稳导致漂移阈值无意义
- 预算太紧，关键时刻仍然被稀释

## 9. 推荐使用场景
- 长视频
- 动作密集视频
- 预算有限但要求高质量标注
- 需要时序敏感 QA 的数据构建
