# 预设管线与文献对照

## 1. 文档目标
本文件不定义新算子，而是说明如何用已经固定好的 10 个算子组织出几条易于汇报、易于实现、易于与文献对齐的预设管线。

## 2. 系统预设

### 2.1 Lite
链路：

`A1(SHOT) -> A2 -> A3(CENTER) -> A6 -> A9(SEGMENT_CAPTION) -> A10`

特点：

- 成本最低
- 便于快速跑通
- 适合短视频和概览型标注

### 2.2 Balanced
链路：

`A1(HYBRID) -> A2 -> A3 + A4 -> A5 -> A6 + A7 + A8 -> A9 -> A10`

特点：

- 作为通用网页混合视频默认方案
- 同时兼顾基础可用性和高质量增强
- 最适合当前系统主汇报

### 2.3 Heavy
链路：

`A1(SEMANTIC/HYBRID) -> A2(strict) -> A4 -> A5 -> A6 + A7 + A8 -> A9 -> A10(strict)`

特点：

- 强调关键片段、对象局部和时间定位
- 更适合高价值数据子集构建
- 成本最高

## 3. 文献风格对照

### 3.1 Video-ChatGPT / Video-LLaVA 风格
推荐链路：

`A1(FIXED/SHOT-lite) -> A3(UNIFORM) -> A9(FRAME_CAPTION/SEGMENT_CAPTION) -> A10`

对照关系：

- 强调基础多帧输入
- 不以复杂结构化证据为主
- 是最容易复现的起点

### 3.2 Panda-70M 风格
推荐链路：

`A1(HYBRID) -> A3(CENTER/UNIFORM) -> A6 -> A9(SEGMENT_CAPTION) -> A10`

对照关系：

- 强调先切出较一致的片段
- 再利用多模态证据生成较高质量描述
- 最后做候选筛选

### 3.3 长视频自适应风格
推荐链路：

`A1(HYBRID) -> A4(FEATURE_DRIFT/MOTION_AWARE) -> A5 -> A8 -> A9 -> A10`

对照关系：

- 对齐 Adaptive Keyframe Sampling、STORM、Video-in-the-Loop、Think with Grounding 等方向
- 重点在预算重分配、关键时刻保留和时间敏感理解

### 3.4 对象中心风格
推荐链路：

`A1 -> A4 -> A7 -> A8 -> A9(SPATIAL_QA/TEMPORAL_QA) -> A10`

对照关系：

- 对齐 VideoRefer Suite、SpaceVLLM、VideoLoom、SlotVTG
- 重点在对象级、区域级和时空联合 grounding

## 4. 为什么同时保留系统预设和文献预设
- 系统预设用于真实实现和默认使用
- 文献预设用于汇报时解释“这套系统如何承接已有研究”

二者并不冲突。前者偏工程交付，后者偏学术对照。

## 5. 当前推荐主线
如果只选择一条对外展示的主链路，推荐使用：

`Balanced`

原因：

- 覆盖 10 个算子中的绝大多数关键能力
- 不会像 Heavy 一样过重
- 比 Lite 更能体现系统设计价值
