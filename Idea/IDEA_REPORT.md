# 研究想法报告

**研究方向**: 大型视频语言模型数据增强 - 程序性视频理解
**生成日期**: 2026-03-30
**想法评估**: 8 个生成 → 6 个通过初步筛选 → 0 个试点（无 GPU）→ 3 个推荐

## 景观总结

当前程序性视频理解领域呈现出从扁平视频描述向结构化层次表示演进的清晰趋势。关键进展包括：(1) 从昂贵的密集标注转向可扩展的弱监督，(2) 层次化任务结构（目标-步骤-子步骤）的兴起，(3) 场景图用于细粒度对齐的集成，(4) 对抗性和反事实数据用于鲁棒性增长的兴趣。

文献揭示了四个关键结构性缺口：(1) 宏观-微观脱节 - 层次化任务结构和场景图被孤立使用；(2) 反事实多样性有限 - 反事实数据专注于状态变化，而非拓扑扰动；(3) 计算复杂度 - 完整图解析是 O(N²)，对长视频不切实际；(4) Video-LLM 幻觉 - 语言先验幻觉由于结构化监督不足。

我们的 MT-MG 方法通过联合宏观任务结构与微观场景图、意图驱动的稀疏化和拓扑反事实扰动来解决这些缺口。

## 推荐想法（按排名）

### 🏆 想法 1: MT-MG 联合表示与意图驱动稀疏化

- **假设**: 通过联合宏观任务树和微观场景图，并使用意图驱动的稀疏化策略，可以显著降低 Video-LLM 的语言先验幻觉，同时保持计算效率。

- **最小实验**: 在 Ego4D 或 COIN 数据集上构建 MT-MG 表示，训练 Video-LLM，与纯文本基线、仅树基线、仅图基线进行比较。评估宏观规划能力（任务预测）、细粒度对齐（时序定位）和幻觉率（事实一致性）。

- **预期结果**: MT-MG + 稀疏化在所有指标上应优于单独使用树或图，幻觉率显著降低（绝对下降 >10%）。

- **新颖性**: 8/10 — 最近工作：Ego4D Goal-Step（宏观）和 EASG（微观）分别存在，但未联合；State-Change Counterfactuals（ICCV 2025）涵盖状态变化但非拓扑扰动。

- **可行性**:
  - 数据：Ego4D、COIN、HowTo100M 可用
  - 计算：需要 4-8 GPU 天用于完整实验
  - 实现：中等复杂度，需要图解析和层次结构建模

- **风险**: MEDIUM — 稀疏化策略可能过滤掉"意图无关但真实"的事件（确认偏差风险）

- **贡献类型**: 方法 + 实证发现

- **试点结果**: SKIPPED: 需要 GPU

- **审稿人可能的反对意见**: "意图驱动的稀疏化可能引入确认偏差，过滤掉重要的背景事件"

- **为什么我们应该做这个**: 这是第一个将宏观任务结构与微观场景图联合的方法，解决了 Video-LLM 幻觉的核心数据瓶颈。

### 想法 2: 拓扑反事实扰动用于硬负样本生成

- **假设**: 通过在图结构层面定义合法的扰动算子（实体替换、状态阻断），生成的硬负样本可以强制 Video-LLM 依赖视觉证据而非语言共现性，从而显著降低幻觉率。

- **最小实验**: 在程序性视频数据集上，使用拓扑反事实生成 QA 对，训练 Video-LLM，与随机文本修改基线比较。评估幻觉率、时序推理准确性和对抗鲁棒性。

- **预期结果**: 拓扑反事实训练的模型幻觉率应显著低于基线（绝对下降 >15%），在对抗性问题上表现更好。

- **新颖性**: 7/10 — 最近工作：State-Change Counterfactuals（ICCV 2025）涵盖状态变化但非拓扑扰动；CaptainCook4D 包含错误执行但未系统化生成反事实。

- **可行性**:
  - 数据：需要程序性视频 + 图解析
  - 计算：需要 2-4 GPU 天
  - 实现：中等复杂度，需要图操作和扰动算子

- **风险**: MEDIUM — 拓扑扰动可能生成不合理的场景，影响训练质量

- **贡献类型**: 方法 + 实证发现

- **试点结果**: SKIPPED: 需要 GPU

- **审稿人可能的反对意见**: "拓扑反事实可能生成语义不合理的场景，影响模型学习"

- **为什么我们应该做这个**: 这是第一个在图结构层面系统化生成硬负样本的方法，超越了传统的文本域随机修改。

### 想法 3: 自适应稀疏化策略与多模态验证

- **假设**: 通过动态调整稀疏化阈值并交叉验证宏观意图与微观证据，可以在保持计算效率的同时减少确认偏差，提高 MT-MG 表示的鲁棒性。

- **最小实验**: 在 MT-MG 框架中实现自适应稀疏化（基于置信度动态调整阈值）和多模态验证（微观证据验证宏观意图），与固定阈值基线比较。评估稀疏化效率、保留事件准确性和下游任务性能。

- **预期结果**: 自适应稀疏化应在保持计算效率的同时，保留更多真实事件（召回率提升 >10%），下游任务性能优于固定阈值。

- **新颖性**: 6/10 — 最近工作：Video-Mined Task Graphs 使用概率图但无稀疏化；现有稀疏化方法多为固定阈值。

- **可行性**:
  - 数据：需要程序性视频 + MT-MG 表示
  - 计算：需要 2-3 GPU 天
  - 实现：中等复杂度，需要置信度估计和动态阈值调整

- **风险**: LOW — 方法相对保守，即使失败也可回退到固定阈值

- **贡献类型**: 方法改进

- **试点结果**: SKIPPED: 需要 GPU

- **审稿人可能的反对意见**: "自适应策略可能增加计算开销，抵消稀疏化的效率优势"

- **为什么我们应该做这个**: 这解决了 MT-MG 中确认偏差风险的核心问题，提高了方法的鲁棒性。

## 关键技术组件

### 宏观树（Macro-Tree）
- 层次结构：目标-步骤-子步骤
- 生成方式：通用大语言模型 + ASR/密集字幕自顶向下解析
- 作用：提供任务与步骤边界，约束微观图解析

### 微观图（Micro-Graph）
- 结构：局部时空场景图
- 生成方式：预训练 VidSGG 模型和跟踪器自底向上抽取
- 作用：捕获细粒度实体交互和状态变化

### 意图驱动稀疏化
- 策略：利用宏观步骤意图过滤微观图中低相关性的冗余边
- 权衡：大幅压缩 O(N²) 图节点规模，但可能过滤"意图外但真实"事件
- 缓解：设置置信度阈值和保留高频背景节点

### 拓扑反事实扰动
- 算子：实体替换、状态阻断等合法图结构修改
- 目标：生成硬负样本，强制模型依赖视觉证据
- 区别：区别于纯文本域的随机修改，在图结构层面操作
